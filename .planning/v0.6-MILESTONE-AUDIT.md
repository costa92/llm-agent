## Milestone properties held

- **Zero-dependency core preserved** — all v0.6 work landed in the
  standalone `llm-agent-rag` (and `llm-agent-otel`); core `llm-agent`
  untouched. `llm-agent-rag` itself added no new module dependency.
- **Seam discipline** — every new capability is an interface
  (`ScoringModel`, `Judge`, `Asker`, `Redactor`, `InjectionScanner`,
  `QueryDecomposer`, `QueryReformulator`) with a bundled implementation,
  tested deterministically with scripted models / stubs.
- **Quality-not-packaging** — no deployment surface added, per the
  milestone's stated scope.

   until `llm-agent-rag` is re-tagged and `otelrag/go.mod`'s `require` is
   bumped. This is a milestone-close step (the v0.5 pattern).

   lexical path and the lexical conformance suite compile and skip clean
   but were never run against a real database (no live DB available).
   Carried forward from v0.5.
3. **Regex-based safety is best-effort** — `guard`'s PII and injection
   detection cover known patterns, not novel/obfuscated attacks; the
   `Rule`/`Pattern` slices are the extension points. A model-based
   classifier is a future-milestone item.
