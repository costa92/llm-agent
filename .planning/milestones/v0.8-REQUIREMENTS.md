## Milestone Scope

v0.8 delivers the **Tier-3 GraphRAG** capabilities v0.7 explicitly deferred
(v0.7 keystone calls KG-1 and KG-6). Building on the shipped v0.7 stack —
the `graph` package, the `store.GraphStore` optional capability, and
`retrieve.GraphRetriever` — v0.8 adds hierarchical community detection,
lazy community summaries, a `rag.System.AskGlobal` map-reduce global-search
answer path, and an opt-in embedding-similarity fuzzy entity-resolution
pre-pass. The milestone is additive: no existing seam changes shape.

Reference: `.planning/research/v0.8-graphrag-tier3-SUMMARY.md` — the
GraphRAG Tier-3 domain research and the KG3-1..KG3-8 keystone decisions.

## v0.8 Requirements

### Community detection and persistence

- [x] **RAG-GRAPH3-01**: the `graph` package gains a `Community` type and a
      `CommunityDetector` seam; a pure-stdlib **Louvain** detector produces a
      deterministic community hierarchy, unit-tested with golden output. A
      `LabelPropagationDetector` alternative ships behind the same seam. No
      LLM, no embedder, no new dependency.
- [x] **RAG-GRAPH3-02**: community persistence sits behind the graph-store
      capability (sibling `CommunityStore` capability) — a pure-stdlib
      in-memory implementation and a `postgres` implementation
      (`_communities` table); a shared conformance suite covers both.
      Community detection runs in Go regardless of store and is wired into
      `Import` as a post-`Canonicalize` stage; a graph-changing
      `ReplaceSource` re-ingest re-detects communities.
- [x] **RAG-GRAPH3-03**: the `graph` package gains a `CommunityReport` type
      and a `CommunitySummarizer` seam; an `LLMCommunitySummarizer` over
      `generate.Model` writes a community report with lenient parsing,
      scripted-model tested. Reports are generated **lazily** and cached
      keyed by a deterministic content hash, persisted via the
      community-store capability (`postgres` `_community_reports` table).
- [x] **RAG-GRAPH3-04**: `rag.System.AskGlobal` answers whole-corpus
      questions by map-reduce over community reports. It does not implement
      `retrieve.Retriever` and does not pass through
      `HybridRetriever`/rerank/pack; `Ask` is unchanged. A global-search
      block surfaces in `Diagnostics`. An opt-in `PrewarmCommunityReports`
      fills the cache eagerly.
- [x] **RAG-GRAPH3-05**: the `graph` package gains an `EntityResolver` seam;
      an `EmbeddingEntityResolver` over `embed.Embedder` clusters
      near-duplicate entities by cosine similarity (same-`Type` only,
      conservative threshold) and rewrites their names — and relation
      endpoints — to a shared surface form **before** `graph.Canonicalize`
      runs. The `rag.System` default is `NoopEntityResolver`.
- [x] **RAG-GRAPH3-06**: the `eval` package gains a global-search
      evaluation harness over the RAG-Triad / `LLMJudge` path (not
      `RunGraphAB` recall@k); a deterministic scripted-model worked example
      ships, and `docs/graphrag.md` is updated with Tier-3 usage, the
      lazy-vs-eager tradeoff, the fuzzy-resolution false-positive caveat,
      and the v0.9 deferral list.

## Out of Scope

| Feature | Reason |
|---------|--------|
| DRIFT search | A refinement on top of working global + local search; deferred to v0.9 |
| Incremental community maintenance | v0.8 does full per-namespace re-detection; incremental maintenance is a deferred optimization |
| Leiden community detection | Marginal gain over Louvain for this SDK's graph sizes; a documented future swap behind the seam |
| Path-ranking / structured subgraph-as-evidence output | Tier-2 path retrieval — a v0.9+ item |
| Claim / covariate extraction | MS-GraphRAG's third primitive — out of scope |
| A dedicated graph database (Neo4j etc.) | Community detection runs in Go regardless of store |
| Embedding or graph-store deps in core `llm-agent` | Violates the zero-dependency core value |
