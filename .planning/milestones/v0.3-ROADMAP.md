# Milestone v0.3: Deployable Multi-Repo Release

**Status:** ✅ SHIPPED 2026-05-12
**Phases:** 0-6
**Total Plans:** 40

## Overview

`v0.3` took `llm-agent` from a stdlib-only, mock-driven learning library to a
deployable multi-repo system with:

- a stable `llm/v2` capability surface in the core repo
- three real provider adapters in `llm-agent-providers`
- OpenTelemetry wrappers in `llm-agent-otel`
- a runnable customer-support reference service in `llm-agent-customer-support`

The milestone intentionally followed a hybrid walking-skeleton shape:
Generate -> Stream -> Tools -> Embeddings across OpenAI, Anthropic, and Ollama
before observability and the reference service were layered on top.

Phase 7 was explicitly removed from the shipped milestone boundary at close.
It remains future work for the deprecation-removal cycle after the required
calendar gate.

## Milestone Summary

**Key Accomplishments**

- Introduced the `llm/v2` capability model without breaking the stdlib-only
  core value of the primary repo.
- Shipped real OpenAI, Anthropic, and Ollama adapters across Generate, Stream,
  Tools, and Embeddings.
- Refactored core agents to negotiate provider capabilities instead of
  assuming them.
- Shipped capability-preserving OTel wrappers with bounded observability
  defaults.
- Delivered a runnable customer-support demo service with RAG, guardrails, and
  trace visibility.

**Key Decisions**

- Hybrid walking-skeleton build order was ratified and held across the entire
  milestone.
- `ProviderInfo` stayed per-bound-model rather than per-provider.
- The core repo remained stdlib-only; dependencies live only in sister repos.
- Kubernetes stayed out of `v0.3`; the local compose demo was treated as the
  production-readiness boundary for this milestone.
- Phase 7 stayed calendar-gated and was not pulled into the `v0.3` closeout.

**Issues Resolved**

- Cross-repo build discipline and release-branch safety gates were established
  before adapter work shipped.
- Provider-native streaming and tool-call differences were normalized behind
  shared conformance contracts.
- The refsvc observability path now proves both end-user trace correlation and
  collector tail-sampling behavior against a live local stack.

**Issues Deferred**

- `DEPRC-01..04` move to the future deprecation-removal cycle after the
  required calendar gate.
- Formal standalone `*-VERIFICATION.md` coverage is still uneven after Phase 0.
- A full compose-native app-container rerun is still environment-sensitive and
  remains optional follow-up rather than a release blocker.

**Technical Debt Incurred**

- Nyquist validation artifacts are still missing for Phases 2-6.
- Demo dashboard panels remain observability-oriented approximations rather
  than billing-grade cost truth.
- Collector metrics endpoint proof was weaker than Tempo-retention proof in the
  closeout environment, so the archived REFSVC-12 evidence relies on direct
  OTLP probes plus Tempo retention checks.
