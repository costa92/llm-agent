## Milestone v0.8: GraphRAG Tier-3 — communities, global search, fuzzy resolution

**Goal**: deliver the Tier-3 GraphRAG capabilities v0.7 deferred — cluster
the entity graph into a hierarchical community structure, summarize
communities lazily into "community reports", answer whole-corpus
"sense-making" questions by map-reduce over those reports, and merge
near-duplicate entities by embedding similarity. Additive — no existing v0.7
seam changes shape, and no new module dependency.

**Repos**: `llm-agent-rag` (only)

**Requirements in scope**: `RAG-GRAPH3-01..06`

**Keystone decisions** (ratified from `.planning/research/v0.8-graphrag-tier3-SUMMARY.md`):

- **KG3-1** — v0.8 targets Tier-3 GraphRAG: hierarchical community
  detection + community summaries + map-reduce global search +
  embedding-similarity fuzzy entity resolution. DRIFT search, path-ranking,
  and claim extraction are out of scope.
- **KG3-2** — community report generation is **lazy by default**
  (LazyGraphRAG): communities detected at ingest, reports generated at query
  time and cached. Eager summarization is a thin opt-in
  (`PrewarmCommunityReports`).
- **KG3-3** — community detection is pure stdlib and store-agnostic: a
  `graph.CommunityDetector` seam with a deterministic stdlib **Louvain**
  default and a `LabelPropagationDetector` alternative. No graph database,
  no new dependency.
- **KG3-4** — global search is a **separate answer path**, not a
  `Retriever`: `rag.System.AskGlobal` does map-reduce over community
  reports, never passing through `HybridRetriever`/rerank/pack.
- **KG3-5** — communities persist behind the graph-store capability — a
  sibling `store.CommunityStore` capability; the v0.7 `GraphStore` interface
  stays byte-identical.
- **KG3-6** — determinism is mandatory: detection deterministic by
  construction; LLM-backed types scripted-model tested; the embedding-backed
  fuzzy resolver scripted-embedder tested; cache keys deterministic.
- **KG3-7** — re-detection on re-ingest from day one: a graph-changing
  re-ingest re-detects communities for that namespace; the report cache is
  invalidated by content-hash mismatch.
- **KG3-8** — fuzzy resolution is an opt-in pre-pass: `EmbeddingEntityResolver`
  rewrites near-duplicate names before `graph.Canonicalize`; the default is
  `NoopEntityResolver` (v0.7 behavior byte-identical).

## Phases

### Phase 23: Community detection and graph-store persistence — complete 2026-05-19

**Requirements covered**: `RAG-GRAPH3-01`, `RAG-GRAPH3-02`

- `23-01` `graph` package: `Community` type + `CommunityDetector` seam;
  pure-stdlib Louvain detector + `LabelPropagationDetector`; golden tests.
- `23-02` `store.CommunityStore` sibling capability + in-memory impl +
  `storetest.RunCommunityConformance`.
- `23-03` `postgres` `CommunityStore` (`_communities` table); community
  detection wired into `Import` with re-detection on re-ingest.

### Phase 24: Community summaries and global search — complete 2026-05-19

**Requirements covered**: `RAG-GRAPH3-03`, `RAG-GRAPH3-04`

- `24-01` `graph` `CommunityReport` + `CommunitySummarizer` seam +
  `LLMCommunitySummarizer` + `CommunityContentHash`; report persistence on
  `CommunityStore` (in-memory + postgres `_community_reports`).
- `24-02` `rag.System.AskGlobal` map-reduce global search; `Diagnostics.Global`.
- `24-03` opt-in `PrewarmCommunityReports`; `GraphTrace.CommunityIDs` local
  attribution; a deterministic worked example.

### Phase 25: Fuzzy entity resolution and evaluation — complete 2026-05-19

**Requirements covered**: `RAG-GRAPH3-05`, `RAG-GRAPH3-06`

- `25-01` `graph.EntityResolver` seam + `NoopEntityResolver` +
  `EmbeddingEntityResolver`; wired as an opt-in pre-pass before
  `Canonicalize` in `Import`.
- `25-02` `eval` global-search harness (`GlobalEvaluator` over the
  Triad/`LLMJudge` path).
- `25-03` `docs/graphrag.md` Tier-3 update + the v0.9 deferral list.

## Outcome

All three phases executed and verified green; milestone audit PASS (6/6
requirements, KG3-1..KG3-8 honored). `llm-agent-rag` gained **no new
dependency** — community detection is pure stdlib, summarization reuses
`generate.Model`, fuzzy resolution reuses `embed.Embedder`. `llm-agent-rag`
tagged `v0.5.0`.
