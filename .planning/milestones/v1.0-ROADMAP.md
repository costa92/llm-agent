## Milestone v1.0: API stabilization and the compatibility promise

**Goal**: lock the public API of `llm-agent-rag`, document a Go-module
compatibility promise, and tag `v1.0.0`. A stabilization / quality
milestone — **no new features**. An audit → fix → verify-and-tag arc:
audit the exported surface, apply the pre-freeze naming/consistency fixes
(free now, impossible after the tag), document every package and the
compatibility policy, add a full-surface API-snapshot gate, and freeze.

**Repos**: `llm-agent-rag` (only)

**Requirements in scope**: `RAG-API-01..06`

**Keystone decisions** (ratified from `.planning/research/v1.0-api-stabilization-SUMMARY.md`):

- **KS-1** — v1.0 scope is `llm-agent-rag` v1.0.0 **only**. The core
  `llm-agent` module and the sister repos stay on their own version
  tracks; the `contract` package is the documented cross-repo seam. The
  only cross-repo touch is documenting that seam in `docs/compatibility.md`.
- **KS-2** — v1.0 is a **freeze, not a feature milestone**. No new
  features, no `eval` redesign, no deployment layer. v0.9's deferred items
  stay deferred to `v1.x`/`v2`.
- **KS-3** — the `ragkit` root package becomes a deliberate, documented
  doc-anchor: keep `doc.go` and the name, rewrite its comment to state it
  is a doc anchor and callers import the sub-packages.
- **KS-4** — pre-freeze naming-consistency fixes are made now or never:
  `eval.Evaluator`/`Result` → `RetrievalEvaluator`/`RetrievalResult` for
  symmetry; the `Ask*`/`*Options` naming is reviewed and ratified as-is.
  Every ruling recorded in writing. No naming change after `v1.0.0`.
- **KS-5** — v1.0 ships a written `docs/compatibility.md`: the Go
  import-compatibility rule (additive-only within v1, interface-method
  gotcha), semver, the `/v2` rule, the `contract` sub-contract, the
  `postgres` dep policy, the `adapter/llmagent` coverage, the Go-version
  floor.
- **KS-6** — a full exported-surface snapshot gate, **stdlib-only**: a
  committed `go doc`/`go/packages`-based snapshot + a `go test`-time
  regeneration-diff, generated once at the freeze as the v1 baseline. It
  complements — does not replace — the cross-repo `contract` gate.
  `apidiff` is rejected as a `go.mod` require.
- **KS-7** — the doc bar is release-blocking: every importable package
  gets a package comment, every exported symbol a name-prefixed comment,
  the stale README is corrected, `CHANGELOG.md` gets a `[v1.0.0]` entry. A
  v1.0 with blank `pkg.go.dev` pages does not ship.
- **KS-8** — hard constraints hold trivially: no new dependency; stdlib +
  the `postgres`/`adapter/llmagent` islands; no `replace` in the release
  branch; `go.sum` stays committed (correct for this module).

## Known Carry-forward Debt

- Live-Postgres CI wiring (testcontainers-go or GH Actions services) is
  still pending from v0.5; the Phase 14/21/23-24 `postgres` paths need
  verification against a live database.
- Incremental community maintenance is deferred (v0.9 KG4-5) — v0.8's full
  re-detection stays.
- The `llm-agent-rag` deployment layer (HTTP service, CLI, caching) remains
  a deliberate non-goal, deferred since v0.6.
- Regex-based content safety (`guard`, v0.6) is best-effort.
- `EmbeddingEntityResolver` (v0.8) has documented false-positive risk.
- The refsvc demo remains intentionally demo-grade.
