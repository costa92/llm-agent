# Requirements: v1.1 — Ecosystem Alignment

**Defined:** 2026-05-21
**Core Value:** the core `llm-agent` module stays stdlib-only and zero-dep;
the umbrella's headline claim — "four coordinated repos plus a standalone
RAG SDK" — is made true at the *release* level: the core and the three
sister repos are brought current with `llm-agent-rag v1.0.0` and each cut
to a coherent, stable, tagged baseline.

## Milestone Scope

v1.1 is an **alignment / housekeeping milestone — no new features**.
`llm-agent-rag` reached a frozen, documented, gate-protected `v1.0.0` on
2026-05-21; v1.0 was deliberately scoped to that one repo (KS-1). Six
milestones later the umbrella is coordinated at the *source* level
(everything builds) but not at the *release* level — every sister `go.mod`
lags. v1.1 closes that gap.

The research audit (`.planning/research/v1.1-ecosystem-alignment-SUMMARY.md`)
confirmed all five repos build and pass their own tests today, and every
cross-repo dependency bump resolves and compiles. The work is concentrated:
the core `llm-agent` `go.mod` pins `llm-agent-rag v0.1.4` (8 minors + a
major stale) and bumping it surfaces **7 facade-test `vector dimension
mismatch` failures** — the one genuine code fix. The rest is branch
landing, a coordinated dependency-bump + re-tag wave, and an umbrella
drift-detection gate.

**Scope is the core `llm-agent` + the three sister repos** (`llm-agent-otel`,
`llm-agent-providers`, `llm-agent-customer-support`). `llm-agent-rag` is
the **fixed point** every repo aligns *to* — its frozen v1.0.0 API is not
touched (KE-2).

The milestone is an **align → land → re-tag → gate** arc. Reference:
`.planning/research/v1.1-ecosystem-alignment-SUMMARY.md` — the per-repo
audit and the KE-1..KE-7 keystone decisions this milestone ratifies.

## v1.1 Requirements

### Core facade re-alignment

- [ ] **ECO-01**: the core `llm-agent` RAG facade is repaired against
      `llm-agent-rag v1.0.0` — `go.mod` bumped to `v1.0.0`, `go.sum`
      refreshed, the 7 `rag: vector dimension mismatch` facade-test
      failures fixed by realigning the facade adapters (`rag/store.go`,
      `rag/embedder.go`, `rag/rag.go`) to the v1.0.0 SDK surface, the
      `rag/contract_test.go` cross-repo gate still compiles, and the core
      is **proven to stay stdlib-only** (`go list -deps ./rag` lists zero
      third-party modules; `go.sum` holds only the `llm-agent-rag` lines).

### Sister-repo branch landing & hygiene

- [ ] **ECO-02**: every sister repo's `main` reflects its true current
      state — `llm-agent-otel`'s `feat/otelrag-wrap-rag-system` (the
      4-commit `otelrag` RAG-wrapping feature) and
      `llm-agent-customer-support`'s `fix/pr-governance-auto-merge-permissions`
      (2 CI-fix commits) are merged to `main`; `llm-agent-providers` is
      confirmed clean on `main` with its untagged deepseek/minimax work
      present and green; stale per-repo governance branches are pruned. No
      repo is tagged in this requirement.

### Coordinated dependency-bump & re-tag wave

- [ ] **ECO-03**: all four repos consume current sibling tags and are cut
      coordinated stable tags in dependency order (core → otel → providers
      → customer-support) — proposed `llm-agent v0.5.0`,
      `llm-agent-otel v0.2.0` (rag dep `v0.3.0`→`v1.0.0`),
      `llm-agent-providers v0.2.0` (captures deepseek/minimax),
      `llm-agent-customer-support v0.2.0` (otel/providers deps → the new
      tags). Every tagged-release branch carries **zero `replace`
      directives**; each repo builds and tests green before its tag is cut.

### Umbrella coherence gate

- [ ] **ECO-04**: the umbrella CI gains a dependency-currency gate — a
      check that fails when a sister `go.mod` pins a sibling version older
      than that sibling's latest tag — closing the `test.yml` (pins the
      stale dep) vs `umbrella.yml` (checks out rag master) blind spot that
      let the drift accumulate silently.
- [ ] **ECO-05**: a full umbrella coherence verification passes — a clean
      5-repo build + test proving every repo resolves the coordinated tag
      set — and the coordinated tag set is recorded; the milestone is
      audited (`.planning/v1.1-MILESTONE-AUDIT.md`).

## Out of Scope

| Feature | Reason |
|---------|--------|
| The `llm-agent-rag` deployment layer (HTTP service, CLI, caching) | A feature milestone, deferred since v0.6; v1.1 is alignment, not feature growth (KE-1) |
| Productionizing the `customer-support` demo | It is intentionally demo-grade; v1.1 aligns its deps only, it does not productionize it |
| Incremental community maintenance | A v0.9 deferral (KG4-5) — additive `v1.x` feature work for a later rag milestone |
| Live-Postgres CI wiring | Genuine carried-forward infra debt, but a CI-capability project of its own size — stays deferred (KE-7) |
| Any change to the `llm-agent-rag` frozen v1.0.0 public API | Post-v1.0 discipline: `v1.x` additive-only, breaking ⇒ `/v2`; rag is the untouched fixed point (KE-2) |
| A new non-stdlib dependency in core `llm-agent` | Absolute hard rule — the facade repair is proven to keep core stdlib-only (KE-3) |
| K8s / Helm packaging | Standing hard-rule non-goal |
