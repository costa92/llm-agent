## Milestone v0.9: GraphRAG refinements — DRIFT search and path-ranking

**Goal**: ship two of the three GraphRAG refinements v0.8 deferred —
path-ranked subgraph evidence and DRIFT hybrid search — completing the
SDK's practical GraphRAG spectrum (lightweight local, path-ranked local,
community global, DRIFT hybrid). Additive — no existing seam changes shape,
no new module dependency.

**Repos**: `llm-agent-rag` (only)

**Requirements in scope**: `RAG-GRAPH4-01..04`

**Keystone decisions** (ratified from `.planning/research/v0.9-graphrag-refinements-SUMMARY.md`):

- **KG4-1** — v0.9 ships DRIFT search and path-ranking / subgraph-as-evidence;
  incremental community maintenance is deferred again. Claim extraction and
  a graph database remain out of scope.
- **KG4-2** — DRIFT is a third answer path, `rag.System.AskDrift`,
  orchestrating the v0.8 global primer and v0.7's local `GraphRetriever`.
  Not a `Retriever`, not a mode flag. Extends KG3-4.
- **KG4-3** — DRIFT's local loop is hard-bounded: round cap (default 2,
  hard cap 3) + termination on no new follow-up entities.
- **KG4-4** — path ranking is deterministic, pure stdlib, an opt-in
  additive mode on `retrieve.GraphRetriever`; `Retrieve` is byte-identical
  when path mode is off.
- **KG4-5** — incremental community maintenance deferred to v1.0+, with a
  documented profiling trigger; v0.8's full re-detection stays.
- **KG4-6** — determinism mandatory: path ranking golden-tested; DRIFT's
  LLM steps scripted-model tested, its orchestration golden-tested.
- **KG4-7** — no new dependency; no existing seam changes shape.
