## Milestone v1.2: Core Capability Deepening

v1.2 is the umbrella's first **core-feature** milestone since v0.3. After
six milestones deepening `llm-agent-rag` (v0.5→v1.0) and one milestone
aligning the ecosystem (v1.1), the core `llm-agent` module has been frozen
on its v0.4-era agent surface for roughly a year. v1.2 ships three new
**agent-runtime governance primitives** — budget/cancellation context,
policy/safety middleware, and a `Supervisor` multi-agent coordinator — on
the existing locked foundation. The 4-candidate research sweep
(`.planning/research/v1.2-core-capability-deepening-SUMMARY.md`) named a
fourth capability — memory scoping (session/project/user) — as a strong
candidate too; this milestone **defers it to v1.3** per KC-2 in favor of
"three capabilities deeply finished" over "four shipped shallow". Theme:
**Core v0.6** — capability additions to core `llm-agent`; memory tiering
deferred to v1.3. An **additive feature milestone — no v2 module path, no
rag re-tag, no sister-repo work**.

**Versioning note (2026-05-21 retarget):** The original ROADMAP scoped the
v1.2 cap tag as `v0.5.1 → v0.6.0`. In practice, **`v0.6.0` was cut early**
2026-05-21 to ship Phase 35 (CC-1 budget) alongside the additive
`agentstest` test-helper sub-package — per semver, adding a new exported
package warrants a minor bump and `v0.6.0` was the natural name. As a
result, the **v1.2 milestone cap tag is retargeted to `v0.7.0`**:

| Phase | Tag at end of phase | Bump type | Why |
|---|---|---|---|
| Phase 35 (CC-1) + `agentstest` | **`v0.6.0` (cut 2026-05-21)** | minor (additive) | new `budget` package + new `agentstest` package |

KC-5 (additive only, no `/v2`) is preserved unchanged — every patch above
is strict-additive (new package only, zero edits to existing public types).
The `v0.7.0` rollup minor is a milestone convention: it gives downstreams
a single tag to pin against post-v1.2 (instead of cherry-picking
`v0.6.0`/`v0.6.1`/`v0.6.2`).

**Requirements in scope**: `CC-1..04`

**Keystone decisions** (ratified from `.planning/research/v1.2-core-capability-deepening-SUMMARY.md`):

- **KC-1** — Multi-agent coordination lives in `orchestrate/` as
  `orchestrate.Supervisor`, **not** a new `agents/coord` package. The
  Supervisor is implemented as a thin facade over `StateGraph[S]` — the
  loop body is a typed state machine (planner-emits-Dispatch →
  worker-runs → planner-observes → repeat-or-finish). This keeps the
  `StateGraph[S]` abstraction working and lets Supervisor inherit its
  cancellation/MaxSteps guard-rails. Workers are `agents.Agent` (so a
  Supervisor can supervise another Supervisor — composition).
- **KC-2** — Memory tiering is **OUT of v1.2 scope**. The 4-candidate
  sweep recommends deferring it. If memory scoping does land later (v1.3
  target), the **decorator shape** (`memory.ScopedStore(inner Memory, scope Scope) Memory`)
  is pre-decided over the interface-field shape: the decorator is purely
  additive (existing `MemoryItem` callers see nothing), and
  `memory.WithScope(ctx, scope)` carries the scope through `ctx`. **This
  keystone is a pre-decision; nothing in v1.2 implements it.**
- **KC-3** — Policy middleware is a `policy` package in core that wraps
  `llm.ChatModel` as a capability-preserving decorator, **mirroring
  `otelmodel.Wrap` (K3)**. It is *not* a hook system. It lives at the
  *model boundary* so every agent paradigm benefits without a code edit.
  Built-in gates: `PIIRedactor` (regex), `InjectionScanner` (regex),
  `MaxInputLen`. Documented composition stack:
  `policy.Wrap(otelmodel.Wrap(provider))` — outer-most policy denies
  before observed; middle observes; inner-most makes the call. Typed
  `Gate` event union (`PreGenerate`/`PostGenerate`/`PreStream`/
  `StreamDelta`/`PostStream`); sentinel `policy.ErrBlocked`.
- **KC-4** — Budget/cancellation is **ctx-keyed for propagation + a
  `budget.Tracker` interface for enforcement** (hybrid). Public surface:
  `budget.WithBudget(ctx, *Tracker) context.Context`, `budget.From(ctx)
  *Tracker` (safe no-op on absent budget), `type Tracker interface{
  Charge(Usage) error; Remaining() Budget }`. Built-in trackers
  `budget.NewStrict` (deny on exhaustion) and `budget.NewSoft` (warn-only).
  **Single integration chokepoint:** the `generateFromPrompt` helper in
  `agents/` calls `budget.From(ctx).Charge(...)` before *and* after the
  LLM call, so every paradigm inherits enforcement uniformly. Cost is
  **opt-in / outside core**: core ships `Budget.Cost float64` plumbing;
  no provider→$ table in core. `MaxSteps` etc. coexist with `Budget.Calls`.
- **KC-5** — Breaking-change avoidance: every new surface is in a *new*
  package or a *new* optional interface. No edit to `llm.ChatModel`,
  `agents.Agent`, `memory.Memory`, or `orchestrate.NodeFunc[S]`. The
  v0.5.1 → v0.6.0 bump is **minor (additive)**; existing v0.5.1 callers
  compile unchanged against v0.6.0. **No `/v2` import path** — v1.2
  deliberately stays under that ceiling.

## Known Carry-forward Debt

Restated verbatim from `.planning/v1.1-MILESTONE-AUDIT.md` "Carry-forward
debt" §, with one v1.2-specific addition for the deferred memory-tiering
candidate:

- **Live-Postgres CI wiring** (from v0.5; KE-7) — `tsvector`, `postgres`
  graph path, `postgres` `_communities`/`_community_reports` paths remain
  env-gated, not runtime-verified against a live database.
- **Incremental community maintenance** (from v0.9, KG4-5) — v0.8's full
  re-detection stays; incremental Louvain is a non-problem at SDK scale.
- **`llm-agent-rag` deployment layer** (HTTP service, CLI, caching) — a
  deliberate non-goal, deferred since v0.6.
- **Regex-based content safety** (`guard`, v0.6 in rag) — best-effort,
  catches known PII + injection patterns, not novel/obfuscated ones.
  **v1.2 inherits this limitation** in the core `policy` package — same
  regex approach (KC-3), same fundamental limit.
- **`EmbeddingEntityResolver` false-positive risk** (v0.8 in rag) —
  ships conservative (high threshold, same-type-only) and opt-in.
- **The refsvc / customer-support demo** stays intentionally demo-grade.
- **Memory scoping (3-layer session / project / user) — deferred to v1.3
  per KC-2.** The reframed shape (`ScopedMemory` decorator + ctx-keyed
  scope propagation via `memory.WithScope(ctx, scope)`) is pre-decided
  in the v1.2 research; v1.3 implements without relitigating the design
  call.
- **Per-sister-repo dependency-currency gate** (v1.1 KE-1 carry-forward)
  — the gate is core-PR-scoped today; broadening it is a v1.3-style
  ecosystem-alignment task.
- **Sister-repo follow-up (post-v1.2)**: the umbrella dep-currency gate
  (KE-6) will fail green when sister repos bump core from v0.5.1 →
  v0.6.0 — that's a future ecosystem-alignment task (v1.3?), **not
  v1.2's work**. Surfaced as a *known* follow-up, not a blocker.
