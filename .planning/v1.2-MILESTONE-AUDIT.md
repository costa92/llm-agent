## Honest deviation record

The v0.7.0 release reframes the originally-planned milestone boundary:

1. **Original plan (STATE.md 2026-05-21):** v0.7.0 = v1.2 close
   (Phase 38), pure milestone-archive tag with no new code; v1.3 =
   future milestone for memory tiering only (`ScopedMemory` decorator
   per KC-2).
2. **Shipped (2026-05-25):** v0.7.0 = v1.2 close + v1.3 memory work
   brought forward with significantly expanded scope (5-pillar
   ChatGPT-style memory system).
3. **Why the deviation:** the operator's session prompt ("缺少 agent 的
   记忆功能，需要参考 chatgpt 的记忆功能来规划设计") triggered a
   design-then-execute pipeline that planned + landed 4 memory PRs
   before the planning sync was rechecked. The deviation was surfaced
   before tag cut (the Plan agent recommended (e4) "pause and let user
   decide"); operator chose (f2) accept-the-merge.
4. **What this audit does:** records the reframe honestly so future
   planning can re-derive both milestones from the shipped code, rather
   than discover an undocumented combined tag later.
