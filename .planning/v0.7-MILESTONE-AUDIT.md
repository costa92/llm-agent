## Keystone-decision compliance (KG-1..KG-7)

- **KG-1** Tier-1 only — community detection / global search absent; listed
  in `docs/graphrag.md` "Deferred to v0.8". ✅
- **KG-2** `GraphStore` is an interface; `go.mod`/`go.sum` diff empty — no
  graph DB. ✅
- **KG-3** dual-mode extraction — `LLMEntityExtractor` +
  `DictionaryEntityExtractor`; the default path stays zero-LLM-capable. ✅
- **KG-4** `GraphRetriever` fuses as a 4th RRF signal gated by
  `EnableGraph`; default `rag.System` behavior unchanged. ✅
- **KG-5** incremental reconciliation shipped in Phase 21 (not deferred) —
  `RemoveGraphBySource` provenance GC + union-merge. ✅
- **KG-6** canonicalization is deterministic exact-match only; fuzzy
  resolution documented as a v0.8 limitation. ✅
- **KG-7** traversal hard-bounded — depth cap 2 enforced in both the
  in-memory walk (`maxGraphDepth`) and the recursive CTE. ✅

## Carry-forward debt

- **Live-Postgres CI wiring** (from v0.5): the `postgres` graph path —
  `*_entities`/`*_relations` tables and the recursive-CTE `Neighborhood` —
  is env-gated, compiles and skips clean, but is **not** runtime-verified
  against a live database. Same standing as the v0.5 `tsvector` debt.
- Fuzzy/embedding entity resolution and MS-GraphRAG community detection are
  deliberate v0.8 deferrals (KG-1, KG-6), not debt.
