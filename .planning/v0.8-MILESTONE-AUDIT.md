## Keystone-decision compliance (KG3-1..KG3-8)

- **KG3-1** Tier-3 scope — detection + summaries + global search + fuzzy
  resolution delivered; DRIFT / path-ranking / claim extraction absent,
  listed in `docs/graphrag.md` "Deferred to v0.9". ✅
- **KG3-2** lazy by default — reports generated at query time, content-hash
  cached; eager `PrewarmCommunityReports` is a thin opt-in. ✅
- **KG3-3** pure-stdlib, store-agnostic detection — Louvain runs in Go;
  Postgres only persists `(entity → community)` rows; `go.mod` diff empty. ✅
- **KG3-4** global search is a separate path — `AskGlobal` never implements
  `retrieve.Retriever`, never calls rerank/pack; `Ask` unchanged. ✅
- **KG3-5** communities behind the graph-store capability — sibling
  `store.CommunityStore`; v0.7 `GraphStore` interface byte-identical. ✅
- **KG3-6** determinism — detection deterministic by construction; LLM-backed
  summarizer/map/reduce scripted-model tested; the fuzzy resolver
  scripted-embedder tested; content hashes deterministic. ✅
- **KG3-7** re-detection on re-ingest — `Import` re-detects the whole
  namespace after a graph-changing re-ingest. ✅
- **KG3-8** fuzzy resolution is an opt-in pre-pass — `EmbeddingEntityResolver`
  rewrites names before `Canonicalize`; `NoopEntityResolver` is the default
  (v0.7 behavior byte-identical). ✅

## Carry-forward debt

- **Live-Postgres CI wiring** (from v0.5): the `postgres` `_communities` and
  `_community_reports` paths — like the v0.5 `tsvector` and v0.7 graph
  paths — are env-gated, compile and skip clean, but are **not**
  runtime-verified against a live database.
- `EmbeddingEntityResolver` has documented false-positive risk; it ships
  conservative (high threshold, same-type-only) and opt-in.
- DRIFT search, incremental community maintenance, and path-ranking are
  deliberate v0.9 deferrals, not debt.
