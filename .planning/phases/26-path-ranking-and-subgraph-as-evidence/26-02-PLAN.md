<objective>
Wire path ranking into `retrieve.GraphRetriever` as an opt-in mode and
surface the ranked paths + structured evidence subgraph on `GraphTrace` /
`rag.Diagnostics`. RAG-GRAPH4-02.
</objective>

<context>
- `graph.PathRanker` / `RankedPath` / `WeightedPathRanker` come from 26-01.
- `retrieve.GraphRetriever.Retrieve` links the query → seed entities,
  `gs.Neighborhood` → `graph.Subgraph` `sub`, emits proximity-decay
  `store.Hit`s + a `GraphTrace`. `retrieve.GraphTrace{SeedEntityIDs,
  ReachedEntityIDs, MaxHop, CommunityIDs}` — `CommunityIDs` (v0.8) is the
  precedent for additive fields.
- `rag.Diagnostics.GraphTrace` is the `retrieve.GraphTrace` value — new
  `GraphTrace` fields surface through `Diagnostics` automatically.
  `VariantRetriever` already propagates the whole `Trace.Graph` block.
- See `26-RESEARCH.md` Decision 3. Path mode OFF must be byte-identical to
  v0.7/v0.8 (KG4-4).
</context>

<tasks>
1. `retrieve/graph.go`:
   - `GraphRetriever` gains a `PathRanker graph.PathRanker` field (nil =
     path mode off).
   - `GraphTrace` gains two additive fields:
     `Paths []graph.RankedPath` and `EvidenceSubgraph *graph.Subgraph`
     (doc them like `CommunityIDs` — populated only in path mode, nil
     otherwise).
   - in `Retrieve`, after `sub` is built and **after** the existing
     chunk-hit / `CommunityIDs` logic (do not disturb it): if
     `r.PathRanker != nil`, build the seed pairs — every unordered pair of
     linked seed entity IDs, in sorted order — call
     `r.PathRanker.RankPaths(sub, pairs)`, and set `trace.Graph.Paths` and
     `trace.Graph.EvidenceSubgraph = &sub`. When `PathRanker` is nil, leave
     both nil — `Retrieve`'s hits, scores, and the rest of the trace are
     untouched.
2. Tests `retrieve/graph_test.go` — extend:
   - path mode on (a `WeightedPathRanker` set): a query linking to two
     entities connected in the graph → `Trace.Graph.Paths` is populated and
     `EvidenceSubgraph` is non-nil; the chunk `[]store.Hit` is identical to
     the path-mode-off run (assert byte-identical hits).
   - path mode off (nil `PathRanker`): `Paths`/`EvidenceSubgraph` are nil —
     the v0.7/v0.8 behavior.
3. Tests `rag/graph_test.go` — extend: an `Ask` (or the graph-retrieval
   path) with a path-mode `GraphRetriever` wired → `Diagnostics.GraphTrace`
   carries the `Paths` and `EvidenceSubgraph`.
4. Run every command in `<verify>`.

Out of scope:
- The worked example, docs — 26-03.
- Changing chunk scoring / RRF fusion — path ranking is extra trace output
  only.
</tasks>
