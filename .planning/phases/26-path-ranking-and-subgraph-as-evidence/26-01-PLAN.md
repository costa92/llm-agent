<objective>
Add path ranking to the `graph` package: a `RankedPath` type, a
`PathRanker` seam, and a deterministic pure-stdlib `WeightedPathRanker` that
enumerates and scores simple multi-hop paths within a `Subgraph`. Pure
`graph` work ŌĆö no store, no LLM, no new dependency. RAG-GRAPH4-01.
</objective>

<context>
- `graph.Subgraph{Entities []Entity, Relations []Relation, Depth
  map[string]int}` ŌĆö the neighborhood result; the path ranker operates on
  it. `Relation` is directed; `Source`/`Target` hold canonical entity IDs;
  `Relation.Weight` is the edge weight.
- `graph/community.go` has an unexported `edgeWeight(r Relation) float64`
  (`Weight` if > 0, else 1.0) ŌĆö reuse it (same package).
- `graph` is a leaf package ŌĆö stdlib + the `generate`/`embed` seams only.
  It cannot import `store`, so the path-length cap is a `graph`-package
  const, not `store`'s `maxGraphDepth`.
- See `26-RESEARCH.md` Decisions 1 and 2. Keystones KG4-4 (deterministic,
  pure stdlib) and KG4-6 (mandatory determinism).
</context>

<tasks>
1. Create `graph/path.go` (package `graph`):
   - `RankedPath struct { EntityIDs []string; RelationIDs []string;
     Score float64 }` ŌĆö `EntityIDs` ordered path[0]..path[n];
     `RelationIDs` the n edges between them.
   - `PathRanker` seam:
     ```go
     type PathRanker interface {
         RankPaths(sub Subgraph, seedPairs [][2]string) []RankedPath
     }
     ```
   - `WeightedPathRanker struct { LengthDecay float64 }` (`LengthDecay <= 0`
     ŌåÆ 0.5) implementing `PathRanker`:
     - a `maxPathLen` const = 2 (edges) ŌĆö the `graph`-package path-length
       cap (matches `store`'s `maxGraphDepth`).
     - for each `[2]string` seed pair, bounded DFS over `sub`'s relations
       (treated undirected for connectivity) enumerating *simple* paths (no
       repeated entity) up to `maxPathLen` edges. Visit neighbors in sorted
       entity-ID order.
     - score each path: `LengthDecay^(edges-1)` ├Ś the product of
       `edgeWeight` over its relations, plus a small bonus when consecutive
       relations share a `SourceChunkID` (provenance overlap). Combine into
       one `Score`.
     - return `[]RankedPath` sorted by `Score` desc, tie-broken by the
       joined `EntityIDs` sequence ŌĆö a total, reproducible order. Drain
       every map to a sorted slice before iterating.
2. Tests `graph/path_test.go`:
   - a fixed `Subgraph` with two entities connected by both a short
     high-weight path and a longer low-weight path ŌåÆ assert the short
     strong path ranks first (golden assertion on `EntityIDs` order and
     relative `Score`).
   - determinism: `RankPaths` on the same input twice ŌåÆ byte-identical.
   - provenance overlap: two paths equal but for shared `SourceChunkIDs` ŌåÆ
     the co-attested one scores higher.
   - no path between a pair ŌåÆ no `RankedPath` for it, no panic; empty
     subgraph / empty seedPairs ŌåÆ empty result.
   - the `maxPathLen` cap: a pair only reachable via 3+ edges yields no
     path.
3. Run every command in `<verify>`.

Out of scope:

- The worked example, docs ŌĆö 26-03.
</tasks>
