  The current `go.mod` pins `llm-agent-rag v0.1.4`.
- Confirmed root cause (`31-RESEARCH.md`): `llm-agent-rag v1.0.0`'s
  `store.InMemoryStore.Search` strictly rejects a query vector whose
  length ≠ the store dimension (`store/inmemory.go:56-57`). The facade's
  `storeAdapter.List` (`rag/rag.go`) emulates enumeration with a
  `nil`-vector wide search `a.inner.Search(ctx, nil, stats.Count)` — a
  `nil` vector is length 0, so under v1.0.0 it fails `ErrDimensionMismatch`,
  mapped by `mapStoreErr` to `rag.ErrDimMismatch`.
- `go get llm-agent-rag@v1.0.0` fetches over SSH — the
  `git config url.insteadOf` + `GOPRIVATE=github.com/costa92/*` rewrite is
  already set on this machine.
- It is EXPECTED that this slice leaves the build green but ~7 facade
  tests failing — that is the diagnosis target, not a slice failure.
- See `31-RESEARCH.md` (root cause + Decision 1).
</context>
