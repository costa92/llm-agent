<objective>
Add community detection to the `graph` package: a `Community` type, a
`CommunityDetector` seam, a deterministic pure-stdlib `LouvainDetector`, and
a `LabelPropagationDetector` alternative. Pure `graph` work — no store, no
LLM, no embedder, no new dependency. RAG-GRAPH3-01.
</objective>

<context>
- `graph.Graph{Entities []Entity, Relations []Relation}`. After
  `Canonicalize`, `Relation.Source`/`Target` hold canonical entity IDs;
  `Relation.Weight` is the edge weight (treat zero as 1.0). `graph` is a
  leaf package — stdlib + the `generate` seam only.
- See `23-RESEARCH.md` Decisions 1 and 2. Keystones KG3-3 (stdlib, dual-mode)
  and KG3-6 (mandatory determinism).
</context>

<tasks>
1. In `graph/graph.go`, add to `Graph` an optional field:
   `Communities []Community` (additive — v0.7 zero-value behavior unchanged).
2. Create `graph/community.go` (package `graph`):
   - `Community struct { ID string; Level int; ParentID string;
     EntityIDs []string; RelationIDs []string }` — `EntityIDs` and
     `RelationIDs` always sorted.
   - `CommunityDetector` seam:
     ```go
     type CommunityDetector interface {
         Detect(ctx context.Context, g Graph) ([]Community, error)
     }
     ```
   - `LabelPropagationDetector struct{}` — deterministic label propagation:
     each entity starts in its own community; iterate entities in sorted
     `ID` order, move each to the community most common among its neighbors
     (tie-break by lowest community ID); repeat until stable or a fixed
     iteration cap. Produces a single level (Level 0).
   - a shared edge-weight helper: `Relation.Weight` when > 0, else 1.0;
     undirected (sum both directions).
   - deterministic community-ID assignment: a helper that, given level +
     sorted member entity IDs, returns a stable ID (e.g.
     `fmt.Sprintf("L%d-%s", level, members[0])`); communities returned
     sorted by ID.
3. Create `graph/louvain.go` (package `graph`):
   - `LouvainDetector struct { Resolution float64 }` (Resolution <= 0 → 1.0)
     implementing `CommunityDetector`.
   - standard Louvain: (a) local modularity-gain greedy moves — iterate
     nodes in sorted ID order, move each to the neighbor community with the
     best positive modularity gain, tie-break by lowest community ID; (b)
     coarsen — collapse each community into a super-node, edges aggregated —
     and repeat. Each coarsening pass yields one hierarchy level: Level 0 =
     finest, higher = communities-of-communities, with `ParentID` linking up.
   - no randomness, no random restarts. Every map is drained into a sorted
     slice before iteration.
4. Tests `graph/community_test.go`:
   - a fixed small graph with two clearly separable clusters bridged by one
     edge → assert `LouvainDetector.Detect` puts each cluster in its own
     Level-0 community (golden assertion on member sets and IDs).
   - determinism: `Detect` the same graph twice → byte-identical `[]Community`.
   - `LabelPropagationDetector` on the same graph → same two-cluster split.
   - hierarchy: a graph of several clusters → assert Louvain emits >1 level
     and `ParentID`s resolve to real higher-level community IDs.
   - empty graph and single-entity graph → no panic, sane output.
5. Run every command in `<verify>`.

Out of scope:
- Persisting communities — 23-02 (`CommunityStore`).
- `Import` wiring — 23-03.
- Leiden — documented future swap behind the seam.
</tasks>
