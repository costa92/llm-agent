<objective>
Add the `store.CommunityStore` optional capability — graph snapshot read +
community persistence — and implement it for the in-memory store, with a
shared conformance suite. Pure stdlib, no postgres edit. RAG-GRAPH3-02.
</objective>

<context>
- `store.GraphStore` is an optional capability (`UpsertGraph` /
  `RemoveGraphBySource` / `Neighborhood` / `FindEntities`); consumers
  type-assert and degrade gracefully. `CommunityStore` is a **sibling**
  capability, same contract — the v0.7 `GraphStore` interface is NOT
  changed (see `23-RESEARCH.md` Decision 3).
- in-memory `GraphStore` lives in `store/graph.go` — `nsGraph` adjacency,
  `InMemoryStore.graphs map[string]*nsGraph`, deterministic ordering.
- `graph.Community` and `graph.Graph` (with its `Communities` field) come
  from 23-01.
- `storetest` — `Factory func(t) store.Store`; `RunGraphConformance` is the
  template for a new `RunCommunityConformance`.
</context>

<tasks>
1. In `store/store.go`, add the capability interface:
   ```go
   // CommunityStore is an optional capability for community detection and
   // persistence. Type-asserted and degraded gracefully, like GraphStore.
   type CommunityStore interface {
       GraphSnapshot(ctx context.Context, namespace string) (graph.Graph, error)
       UpsertCommunities(ctx context.Context, namespace string, communities []graph.Community) error
       Communities(ctx context.Context, namespace string) ([]graph.Community, error)
   }
   ```
2. Create `store/community.go` (package `store`) — implement `CommunityStore`
   on `*InMemoryStore`:
   - `GraphSnapshot` — reconstruct a `graph.Graph` (entities + relations)
     from the namespace's `nsGraph`; deterministic order (sorted by ID);
     unknown namespace → empty `graph.Graph`, no error.
   - `UpsertCommunities` — replace-all: store a deep copy of the community
     slice for the namespace (a new `communities map[string][]graph.Community`
     field on `InMemoryStore`, initialized in `New`).
   - `Communities` — return a deep copy, sorted by community ID; unknown
     namespace → nil, no error.
   - `var _ CommunityStore = (*InMemoryStore)(nil)`.
3. In `store/storetest/storetest.go`, add `RunCommunityConformance(t,
   factory)` (mirror `RunGraphConformance`): skip if the store is not a
   `CommunityStore`; otherwise exercise — `UpsertGraph` then `GraphSnapshot`
   round-trips entities/relations; `UpsertCommunities` then `Communities`
   round-trips a community hierarchy unchanged; a second `UpsertCommunities`
   replaces (not appends); namespaces are isolated; unknown namespace is
   empty/no-error.
4. Tests `store/community_test.go` — run `storetest.RunCommunityConformance`
   against the in-memory store; plus a direct test that a detected hierarchy
   (build one with `graph.LouvainDetector` from 23-01) survives an
   `UpsertCommunities` → `Communities` round-trip byte-identically.
5. Run every command in `<verify>`.

Out of scope:
- postgres `CommunityStore` — 23-03.
- `Import` wiring / `Options.CommunityDetector` — 23-03.
</tasks>
