<objective>
Implement `store.CommunityStore` for the postgres store, and wire community
detection into `rag.Import` as a post-persist stage with re-detection on
re-ingest. Completes RAG-GRAPH3-02 and Phase 23.
</objective>

<context>
- `store.CommunityStore` (23-02) — `GraphSnapshot` / `UpsertCommunities` /
  `Communities`. `postgres.Store` already implements `store.GraphStore`
  (`postgres/graph.go`, `_entities`/`_relations` tables) and `store.Store`;
  `Migrate()` holds idempotent DDL guarded by `isSafeIdent`.
- `rag.Import` (`rag/import.go`) — after the `if persistGraph` block the
  namespace graph is fully persisted (post-`UpsertGraph`, and
  post-`RemoveGraphBySource` on a `ReplaceSource` re-ingest).
  `Options.EntityExtractor`, `ImportResult.Graph *graph.Graph`.
- `graph.CommunityDetector` / `LouvainDetector` from 23-01.
- See `23-RESEARCH.md` Decisions 3 and 4. Keystone KG3-7 (re-detection on
  re-ingest from day one).
</context>

<tasks>
1. `postgres/community.go` (package `postgres`) — implement `CommunityStore`
   on `*Store`:
   - `GraphSnapshot` — `SELECT` all rows of `<table>_entities` and
     `<table>_relations` for the namespace, rebuild a `graph.Graph`,
     deterministic order.
   - `UpsertCommunities` — replace-all in one transaction: `DELETE` the
     namespace's rows from `<table>_communities`, then bulk-insert the new
     set.
   - `Communities` — `SELECT` the namespace's rows, ordered by community ID.
   - `var _ store.CommunityStore = (*Store)(nil)`.
2. In `postgres/postgres.go` `Migrate()`, add the `<table>_communities`
   table (idempotent `CREATE TABLE IF NOT EXISTS`, `isSafeIdent`-guarded):
   columns `namespace text`, `community_id text`, `level int`,
   `parent_id text`, `entity_ids text[]`, `relation_ids text[]`; primary key
   `(namespace, community_id)`.
3. `rag/options.go` — add `CommunityDetector graph.CommunityDetector` to
   `Options`. `rag/system.go` — carry it as `System.communityDetector`.
4. `rag/import.go` — after the existing `if persistGraph` block: if the
   store is a `store.CommunityStore` AND `s.communityDetector != nil`:
   - `snap, err := cs.GraphSnapshot(ctx, opts.Namespace)`
   - `communities, err := s.communityDetector.Detect(ctx, snap)`
   - `cs.UpsertCommunities(ctx, opts.Namespace, communities)`
   - if `res.Graph != nil`, set `res.Graph.Communities = communities`.
   Errors wrap as `fmt.Errorf("rag: detect communities: %w", err)`. Because
   the snapshot is read after reconciliation, a `ReplaceSource` re-ingest
   re-detects the whole namespace automatically (KG3-7). A store that is not
   a `CommunityStore`, or no detector configured, degrades gracefully — no
   detection, no error.
5. `postgres/postgres_conformance_test.go` — wire
   `storetest.RunCommunityConformance` for the postgres store, env-gated
   exactly like the existing `RunGraphConformance` wiring.
6. `rag/community_test.go` — with the in-memory store + a `LouvainDetector`
   in `Options`: import documents, assert `ImportResult.Graph.Communities`
   is populated and the store's `Communities(ns)` matches; then
   `ReplaceSource` re-ingest one document and assert the community set is
   re-detected (reflects the new graph), not stale or appended.
7. Run every command in `<verify>`.

Out of scope:
- Community summaries / reports — Phase 24.
- Global search — Phase 24.
- Live-Postgres verification — env-gated, carried-forward debt.
</tasks>
