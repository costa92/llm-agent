<objective>
Reranking today is heuristic-only (`lexicalBoost`). This slice adds a
model-based reranker behind the existing `rerank.Reranker` interface: an
abstract `ScoringModel` seam, a `ModelReranker` that reranks by model score,
and a concrete `HTTPScoringModel` that calls an external cross-encoder /
hosted rerank API over `net/http`.
</objective>

<context>
- 15-01 added `rerank.RerankScore`, `rerank.Trace.Scores`, and the unexported
  `buildScores(input, output []store.Hit) []RerankScore` helper. `ModelReranker`
  must reuse `buildScores` so its output carries the same explainability data.
- `rerank.Reranker` interface: `Rerank(ctx, Request) ([]store.Hit, Trace,
  error)`. `Request{Query string, Hits []store.Hit}`.
- `HeuristicReranker` builds a per-hit document string from
  `Title + Content + Heading + SectionPath`. `ModelReranker` should build a
  comparable document string so the model sees the same structured text.
- **No new dependency.** A rerank API client is plain `net/http` +
  `encoding/json`. `llm-agent-rag` stays stdlib-only outside `postgres`; no
  build tag is needed. See `15-RESEARCH.md` Decisions 1-3.
- The project's determinism discipline (`ScriptedLLM`): `ModelReranker` is
  tested with a stub `ScoringModel`; `HTTPScoringModel` is tested against an
  `httptest.Server`. No live network in tests.
</context>

<tasks>
1. In `rerank/rerank.go`, add the scoring seam and error:
   ```go
   // ScoringModel scores how well each document answers the query. The
   // returned slice is parallel to documents (one score per document).
   type ScoringModel interface {
       Score(ctx context.Context, query string, documents []string) ([]float64, error)
   }

   var ErrScoringModelRequired = errors.New("rerank: scoring model required")
   ```
2. Add `ModelReranker`:
   ```go
   type ModelReranker struct {
       Model ScoringModel
       TopN  int // if > 0, truncate reranked output to TopN
   }
   ```
   `Rerank`:
   - if `Model == nil` â†’ return `ErrScoringModelRequired`.
   - build one document string per hit (same composition as
     `HeuristicReranker`: title, content, heading, section path).
   - call `Model.Score(ctx, req.Query, docs)`; if the returned slice length
     != len(hits), return a descriptive error.
   - set each hit's `Score` to its model score; sort descending, stable,
     tie-broken by original order.
   - if `TopN > 0` and `len(out) > TopN`, truncate.
   - return `out` and a `Trace` with `InputChunkIDs`, `OutputChunkIDs`, and
     `Scores: buildScores(req.Hits, out)`.
3. Create `rerank/httpmodel.go` with `HTTPScoringModel`:
   ```go
   type HTTPScoringModel struct {
       Endpoint string        // required: full URL of the rerank endpoint
       Model    string        // optional: model name sent in the request
       Token    string        // optional: bearer token (Authorization header)
       Client   *http.Client  // optional: nil -> http.DefaultClient
   }
   ```
   `Score`:
   - error if `Endpoint == ""`.
   - POST JSON `{"model":<Model>,"query":<query>,"documents":[<doc>...]}`.
   - set `Content-Type: application/json`; if `Token != ""`, set
     `Authorization: Bearer <Token>`.
   - non-2xx â†’ error including the status code.
   - decode `{"results":[{"index":<int>,"relevance_score":<float>}...]}`;
     build a `[]float64` of len(documents), placing each result's
     `relevance_score` at its `index`. Indexes out of range â†’ error.
   - return the parallel score slice.
4. Tests:
   - `rerank/rerank_test.go` â€” `ModelReranker` with a stub `ScoringModel`
     that returns scores reversing the input order: assert the output is
     reversed, `Trace.Scores` deltas are populated, and `TopN` truncates.
     Assert `ModelReranker{}` (nil model) returns `ErrScoringModelRequired`.
   - `rerank/httpmodel_test.go` â€” `HTTPScoringModel` against an
     `httptest.Server` that returns a known `results` payload (deliberately
     out of index order): assert `Score` returns scores in document order.
     Assert empty `Endpoint` errors, and a 500 response errors.
5. Run every command in `<verify>`.

Out of scope:
- Wiring `ModelReranker` as the `rag.System` default â€” the default stays

  `rag.Options.Reranker`, which already exists.
- Vendor-specific SDKs or auth flows beyond a bearer token.
- Batching / pagination of very large hit sets.
- Streaming rerank.
</tasks>
