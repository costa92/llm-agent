<objective>
Add a DRIFT-search evaluation harness to `eval` — run whole-corpus
questions through `AskDrift` and score each answer with the RAG-Triad
`Judge`. RAG-GRAPH4-04.
</objective>

<context>
- `rag.System.AskDrift(ctx, q, DriftOptions) (Answer, error)` (27-01);
  `Answer.Diagnostics.Drift` carries `PrimerCommunityIDs`, `Rounds`,
  `RoundEntityIDs`, `ConsultedReports`.
- `eval/global.go` is the direct template: `GlobalAsker` interface,
  `GlobalEvaluator{Asker, Judge, MaxCommunities}`, `GlobalEvalResult`,
  `Run` — it scores `AskGlobal` answers with the `Judge` using
  `Diagnostics.Global.ConsultedReports` as the grounding context.
- `eval` — `Dataset`/`Example`, `LLMJudge` →
  `Judgement{Groundedness, AnswerRelevance}`. DRIFT has no gold chunk set —
  no `RunGraphAB` recall@k.
- See `27-RESEARCH.md` Decision 4.
</context>

<tasks>
1. Create `eval/drift.go` (package `eval`) — mirror `eval/global.go`:
   - `DriftAsker` interface:
     `AskDrift(ctx, question string, opts rag.DriftOptions) (rag.Answer, error)`.
   - `DriftEvalResult struct` — `MeanGroundedness`, `MeanAnswerRelevance`,
     `Examples int`, and a `PerExample` detail slice (question, answer,
     judgement, primer community IDs, rounds run).
   - `DriftEvaluator struct { Asker DriftAsker; Judge Judge;
     MaxCommunities int; Rounds int }` with
     `Run(ctx, dataset Dataset) (DriftEvalResult, error)`:
     - error if `Asker` or `Judge` is nil.
     - per `Example`: `AskDrift` with `DriftOptions{Namespace:
       ex.Namespace, MaxCommunities, Rounds}`; build the judge context by
       concatenating `Answer.Diagnostics.Drift.ConsultedReports` summaries;
       `Judge.Judge` for groundedness + answer-relevance; accumulate means.
     - the gold chunk/doc fields of `Example` are unused — document that.
2. Tests `eval/drift_test.go` — a CI gate mirroring `eval/global_test.go`:
   a scripted `generate.Model` (drives the primer map + local rounds +
   synthesis inside `AskDrift`) and a scripted `Judge`; an in-memory store
   + graph + communities; assert `DriftEvaluator.Run` scores every example,
   the means are in `[0,1]`, the judge receives the consulted-report
   context, and a nil-`Asker`/nil-`Judge` errors.
3. Run every command in `<verify>`.

Out of scope:
- The worked example, docs — 27-03.
</tasks>
