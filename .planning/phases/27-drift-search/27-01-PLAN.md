<objective>
Add `rag.System.AskDrift` — DRIFT hybrid search: a global primer pass, a
hard-bounded local follow-up loop, and a synthesis step. A third answer
path alongside `Ask` and `AskGlobal`. RAG-GRAPH4-03.
</objective>

<context>
- `rag/global.go` has unexported, in-package-reusable pieces:
  `selectCommunities(ctx, cs, ns, communities, question, max)`,
  `s.communityReports(ctx, cs, ns, selected)`, `globalMapPrompt`,
  `parseGlobalMap`, `globalPartial`. `AskDrift` reuses them for the primer —
  no `AskGlobal` refactor.
- `store.CommunityStore` — `Communities`; `store.GraphStore` —
  `Neighborhood(ns, seedIDs, depth)`, `FindEntities(ns, names)`.
  `graph.Community.EntityIDs` are a community's members.
- `Answer{Text, Hits, Prompt, Citations, Diagnostics, Trace}`;
  `Diagnostics.Global` is the template for a `Diagnostics.Drift` block.
  `AskGlobal` installs a fresh `obs.Counter` — mirror that.
- `pack` assembles chunks into prompt context.
- See `27-RESEARCH.md` Decisions 1-3. Keystones KG4-2 (separate path),
  KG4-3 (hard-bounded loop).
</context>

<tasks>
1. `rag/options.go` — add `DriftOptions struct { Namespace string;
   MaxCommunities int; Rounds int; TopK int }` (`MaxCommunities <= 0` → 8;
   `Rounds <= 0` → 2, `> 3` → 3; `TopK <= 0` → 8).
2. `rag/system.go` — add `DriftDiagnostics struct { PrimerCommunityIDs
   []string; Rounds int; RoundEntityIDs [][]string; ConsultedReports
   []graph.CommunityReport }` and an additive `Diagnostics.Drift
   DriftDiagnostics` field.
3. Create `rag/drift.go` (package `rag`) — `func (s *System) AskDrift(ctx,
   question string, opts DriftOptions) (Answer, error)`:
   - `s.model == nil` → `ErrModelRequired`. Install a fresh `obs.Counter`.
   - type-assert `s.store` for `store.CommunityStore` + `store.GraphStore`;
     if the community capability is absent or the namespace has no
     communities, the primer is empty — degrade to a pure local answer (or
     an empty `Answer`), no error.
   - **Primer**: `selectCommunities` → `s.communityReports` → the map step
     (one `globalMapPrompt` generation per report → scored partials, via
     `parseGlobalMap`). Record the selected community IDs + reports. The
     highest-scoring communities' `EntityIDs` are the round-0 seed entity
     IDs (sorted, deduped).
   - **Local follow-up loop**, for round in `[0, Rounds)`:
     - `gs.Neighborhood(ns, seedEntityIDs, 1)` → `graph.Subgraph`; collect
       the subgraph entities' provenance chunks, `s.store.Get` each (skip
       `store.ErrNotFound`), `pack` up to `TopK` into context;
     - `s.model.Generate` with the question + context → a partial answer
       and a leniently-parsed marked list of follow-up entity names;
     - `gs.FindEntities(ns, followupNames)` → next round's seed IDs.
     - **Terminate** on the first of: round cap reached; no new follow-up
       entities; no new reachable entities. Record per-round seed IDs.
   - **Synthesis**: one `s.model.Generate` synthesizing the primer partials
     + every round's partial into `Answer.Text`.
   - return `Answer` with `Diagnostics.Drift` populated (primer community
     IDs, rounds run, per-round entity IDs, consulted reports) and
     `Diagnostics.Metrics` (counter counts, stage timings).
4. Tests `rag/drift_test.go` — a scripted `generate.Model` keyed so primer
   map, each local round, and synthesis are distinguishable; an in-memory
   store with a graph + detected communities:
   - `AskDrift` runs the primer, the bounded loop, and synthesis; the final
     `Answer.Text` is the synthesis output; `Diagnostics.Drift` records the
     primer communities and per-round entity IDs.
   - the loop terminates early when the scripted model emits no follow-up
     entities — `Diagnostics.Drift.Rounds` is below the cap.
   - the round cap holds when the model keeps emitting follow-ups —
     `Rounds` never exceeds 3.
   - a non-`CommunityStore` store, and a namespace with no communities →
     graceful (local-only or empty `Answer`), no error.
5. Run every command in `<verify>`.

Out of scope:

- The worked example, docs — 27-03.
</tasks>
