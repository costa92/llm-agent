<objective>
Close the v0.7 milestone: add graph-retrieval evaluation (a graph-on/off
A/B through the existing metrics harness), a deterministic worked example,
and the GraphRAG docs. This is RAG-GRAPH-06.
</objective>

<context>
- `eval` package — `Dataset`/`Example` (labelled `GoldDocIDs`/
  `GoldChunkIDs`), `Evaluator`, `Metrics` (precision@k / recall@k / MRR /
  grounding@k), the metric helpers (`countMatches`, `firstGoldRank`,
  `anyOverlap`), and `TriadEvaluator`.
- 22-01/02 delivered `GraphRetriever` and the `EnableGraph` toggle on
  `rag.SearchOptions` / `retrieve.Request`.
- `examples/` holds deterministic, scripted-model examples (the project's
  example discipline — no live calls).
- `docs/` holds package docs.
- See `22-RESEARCH.md` Decision 5.
</context>

<tasks>
1. Create `eval/graph.go` — a graph-retrieval A/B:
   - a function that runs a `Dataset` through retrieval **twice** — once
     with `EnableGraph` false, once true — and returns the retrieval
     `Metrics` for each, plus the recall (and MRR) delta. Reuse the
     existing metric machinery; do not fork it.
   - shape it so a caller can answer "does the graph signal improve
     recall on this dataset?" — e.g. `GraphABResult{GraphOff, GraphOn
     Metrics; RecallDelta, MRRDelta float64}` and a
     `RunGraphAB(ctx, ...) (GraphABResult, error)`.
2. Tests `eval/graph_test.go` — a CI gate: build a `rag.System` with the
   `DictionaryEntityExtractor` + the in-memory `GraphStore` + a
   `GraphRetriever`-wired `HybridRetriever`, a seed corpus + dataset where
   the gold answer is reachable through the graph, and assert
   `RunGraphAB` runs both arms and that graph-on recall is `>=` graph-off
   (the graph signal does not regress recall). Deterministic — scripted
   model + `DictionaryEntityExtractor`.
3. Create `examples/graphrag_example_test.go` — a deterministic
   `Example`-style test: import documents with a `DictionaryEntityExtractor`,
   wire a `GraphRetriever` into the system, `Ask` with `EnableGraph: true`,
   and show the graph diagnostics. Scripted model, no live calls.
4. Create `docs/graphrag.md` — the GraphRAG usage doc: how to wire an
   `EntityExtractor`, a `GraphStore`, and a `GraphRetriever`; the
   `EnableGraph` toggle; what v0.7 ships (Tier-1 lightweight GraphRAG); and
   the **explicit v0.8 deferral list** — community detection, community
   summaries, global/DRIFT search, fuzzy entity resolution — plus the note
   that Neo4j is a future `GraphStore` implementation enabled by the
   interface.
5. Run every command in `<verify>`.

Out of scope:
- Community detection, global search, fuzzy resolution — v0.8 (documented
  as deferred, not built).
</tasks>
