<objective>
Add the graph-traversal retriever: an `EntityLinker` seam, a
`LexicalEntityLinker`, and a `GraphRetriever` that links a query to seed
entities, traverses their bounded neighborhood, and returns the provenance
chunks scored by graph proximity. First half of RAG-GRAPH-05.
</objective>

<context>
- `retrieve.Retriever` — `Retrieve(ctx, Request) ([]store.Hit, Trace,
  error)`. `GraphRetriever` implements it.
- `store.GraphStore` (Phase 21) — `FindEntities(ns, names)`,
  `Neighborhood(ns, seedIDs, depth) (graph.Subgraph, error)`;
  `graph.Subgraph{Entities, Relations, Depth map[string]int}`.
- `graph.Entity.SourceChunkIDs` is the provenance — the bridge from a
  graph node to `store.Hit`s. `store.Store.Get(ctx, id)` fetches one
  `StoredChunk`.
- `EntityLinker` must live in `retrieve` (not `graph`): `Link` references
  `store.GraphStore`, and `store` already imports `graph` — `graph` must
  not import `store`. `retrieve` imports both, no cycle.
- See `22-RESEARCH.md` Decisions 1 and 2.
</context>

<tasks>
1. Create `retrieve/graph.go` (package `retrieve`):
   - `EntityLinker` seam:
     ```go
     type EntityLinker interface {
         Link(ctx context.Context, query, namespace string, gs store.GraphStore) ([]graph.Entity, error)
     }
     ```
   - `LexicalEntityLinker struct{}` — `Link` splits the query into
     whitespace tokens (and the whole query), calls `gs.FindEntities` with
     those candidate names, dedups the returned entities by ID. No
     embedder — keeps the default path zero-LLM.
   - `GraphTrace struct { SeedEntityIDs, ReachedEntityIDs []string; MaxHop int }`.
   - `GraphRetriever`:
     ```go
     type GraphRetriever struct {
         Linker   EntityLinker
         Store    store.Store // type-asserted for store.GraphStore
         MaxDepth int          // default 1, hard cap 2
         HopDecay float64      // proximity decay per hop, default 0.5
     }
     ```
     `Retrieve`:
     - type-assert `Store` for `store.GraphStore`; if absent, return
       `nil, Trace{}, nil` (graceful — graph contributes nothing).
     - nil `Linker` → default `LexicalEntityLinker{}`.
     - `Link` the query → seed entities; no seeds → empty result.
     - resolve `MaxDepth` (`<= 0` → 1; `> 2` → 2) and `HopDecay`
       (`<= 0` → 0.5).
     - `Neighborhood(req.Namespace, seedEntityIDs, depth)` → `graph.Subgraph`.
     - for each reached entity, for each `SourceChunkIDs`: `Store.Get` the
       chunk; score `HopDecay^hop` (the entity's `Depth`). A chunk reached
       via several entities keeps its best (lowest-hop) score. A `Get`
       returning `store.ErrNotFound` is skipped, not fatal.
     - sort hits by score desc (tie-break by chunk ID), truncate to
       `req.TopK`.
     - return a `Trace` with a `Graph GraphTrace` block recording the seed
       and reached entity IDs and the max hop reached.
2. Tests `retrieve/graph_test.go`:
   - Build an in-memory `store.Store` (a `store.GraphStore`), `Upsert`
     chunks and `UpsertGraph` a small graph whose entities' provenance
     points at those chunks.
   - `LexicalEntityLinker.Link` resolves a query token to the matching
     entity.
   - `GraphRetriever.Retrieve` from a query linking to a seed entity →
     returns the seed's provenance chunks; a 1-hop neighbor's chunks score
     below the seed's (proximity decay); `Trace.Graph` records seeds and
     reached entities.
   - `GraphRetriever` over a store that is not a `GraphStore` → empty
     result, no error.
