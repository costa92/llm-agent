<objective>
Build the v1.0 exported-surface snapshot gate: a pure-stdlib generator
(`go/parser` + `go/ast` + `go/token` + `go/printer`) that captures
`llm-agent-rag`'s entire exported API as a deterministic text file, the
committed baseline `api/v1.snapshot.txt`, and a `go test` that fails any
PR which renames, removes, or re-signs an exported symbol. RAG-API-05,
KS-6.
</objective>

<context>
- KS-8: **no new dependency.** The generator uses only stdlib:
  `go/parser`, `go/ast`, `go/token`, `go/printer`, `os`,
  `path/filepath`, `sort`, `strings`, `bytes`. `golang.org/x/tools`
  (`go/packages`, `apidiff`) is **rejected** — it is not stdlib.
- The gate **complements** the existing `contract/contract_test.go`:
  `contract` is the narrow cross-repo compile-pin (the core-`llm-agent`
  facade subset); this snapshot is the whole-surface intra-repo diff. Both
  are `go test`-time, both stdlib. Document this in the generator's
  package comment.
- `internal/` is non-importable externally — the gate machinery is
  correctly not part of the public surface. The generator **skips**
  anything under `internal/` and every `_test.go` file.
- `go/parser` parses source regardless of build tags, so the generator
  parses `adapter/llmagent/*.go` directly — the build-tagged adapter **is**
  in the snapshot.
- See `30-RESEARCH.md` Decision 1 for the output-format sketch.
- This slice runs after Phases 28-29 — the surface it captures is the
  frozen, fully-documented one.
</context>

<tasks>
1. Create `internal/apisnapshot/apisnapshot.go` (package `apisnapshot`):
   - `func Generate(moduleRoot string) (string, error)` — walk
     `moduleRoot` with `filepath.WalkDir`; for every `.go` file that is
     **not** a `_test.go` and **not** under an `internal/` path segment,
     parse it with `parser.ParseFile` (mode `parser.SkipObjectResolution`
     is fine; comments not needed).
   - Group declarations by package import path
     (`github.com/costa92/llm-agent-rag/<rel-dir>`; the root dir →
     the module path itself).
   - Collect, for each package, every **exported** declaration:
     - `func` declarations with no receiver → funcs;
     - `func` declarations **with** a receiver whose base type is exported
       → methods, grouped under the receiver type;
     - `type` specs that are exported → types; for a `struct` type, its
       exported fields; for an `interface` type, its exported (and
       embedded) methods;
     - exported `var` / `const` value specs.
   - Render each declaration deterministically via `go/printer` (strip
     function bodies — render signatures only; for a `FuncDecl` set
     `Body = nil` before printing). Field/method/parameter types render
     exactly as written.
   - **Sort everything**: packages by import path; symbols within a
     package by kind then name; struct fields and interface methods by
     name. No map-iteration order may leak into the output.
   - Return the assembled text with a fixed header line
     (`# llm-agent-rag v1 exported API snapshot — generated, do not hand-edit.`).
   - The package doc comment explains the gate and how it complements
     `contract` (see <context>).
2. Create `internal/apisnapshot/apisnapshot_test.go`:
   - a `var update = flag.Bool("update", false, "rewrite api/v1.snapshot.txt")`;
   - `TestAPISnapshot` — resolve the module root (the test file is at
     `internal/apisnapshot/`, so module root is `../..`; resolve to an
     absolute path), call `Generate`, and:
     - with `-update`: write the result to `<root>/api/v1.snapshot.txt`;
     - otherwise: read the committed `api/v1.snapshot.txt` and compare. On
       mismatch, `t.Fatalf` with a readable diff (first differing lines)
       and the exact regeneration command:
       `go test ./internal/apisnapshot/ -run TestAPISnapshot -update`.
   - a `TestGenerateIsDeterministic` — call `Generate` twice, assert the
     two outputs are byte-identical.
3. Generate the baseline: run
   `GOWORK=off GOCACHE=/tmp/go-build go test ./internal/apisnapshot/ -run TestAPISnapshot -update`,
   then inspect `api/v1.snapshot.txt` — confirm it contains every
   importable package (cross-check against `go list ./...`) and
   `adapter/llmagent`, and that known symbols are present
   (`rag.System`, `eval.RetrievalEvaluator`, `graph.WeightedPathRanker`,
   `store.Store`, `adapter/llmagent`'s `ModelAdapter`).
4. Run every command in `<verify>`.

Out of scope:
- Wiring CI / the `-tags llmagent` step — slice 30-02.
- The `CHANGELOG.md` entry — slice 30-03.
- The `v1.0.0` tag — cut at milestone-close by the operator.
</tasks>
