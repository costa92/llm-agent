    **Version:** v0.1 (Phase 1 — Generate-only contract)

    ## 2. The Contract

    Your adapter implements `llm.ChatModel`:

    ```go
    type ChatModel interface {
        Generate(ctx context.Context, req Request) (Response, error)
        Stream(ctx context.Context, req Request) (StreamReader, error)
        Info() ProviderInfo
    }
    ```

    Source: [`llm/chatmodel.go`](llm/chatmodel.go).

    Phase 1 boundary: `Stream` returns a "not implemented" error. The interface
    method exists so the type system stays stable across Phases 1→2.

    The shapes you produce/consume:

    - **Request** — `llm.Request` ([llm/types.go](llm/types.go))
    - **Response** — `llm.Response` ([llm/types.go](llm/types.go))
    - **ProviderInfo** + **Capabilities** — ([llm/info.go](llm/info.go))
    - **Errors** — `llm.AuthError`, `llm.RateLimitError`,
      `llm.InvalidRequestError`, `llm.TransientError` ([llm/errors.go](llm/errors.go))

    Reference shapes for "what does an adapter look like" before writing:

    - [`llm/scripted.go`](llm/scripted.go) — full-capability mock (every
      capability interface satisfied)
    - [`llm/chat_only_mock.go`](llm/chat_only_mock.go) — minimum-viable
      `ChatModel`-only mock; this is exactly the shape your Phase-1 adapter
      should structurally match (plus real HTTP)

    ## 3. The Generate Contract

    Required behavior of `Generate(ctx, req) (Response, error)`:

    | Aspect | Required behavior |
    |--------|-------------------|
    | `req.Messages` | Honored verbatim; mapped to provider's role-tagged messages array |
    | `req.SystemPrompt` | Lifted to provider's system-prompt home: top-level for Anthropic; role:"system" message for OpenAI / Ollama |
    | `req.MaxOutputTokens` (if > 0) | Passed through as the provider's max-tokens parameter |
    | `req.Temperature` (if non-nil) | Passed through if the provider supports it |
    | `req.Tools` | **Phase 1: NOT honored.** `Capabilities.Tools = false`. (See §7.) |
    | `Response.Provider` | Set to the canonical provider name (e.g., "openai"). |
    | `Response.Model` | Set to the bound model from construction. |
    | `Response.Usage.Source` | `llm.UsageReported` if the wire format provides token counts; `llm.UsageUnknown` otherwise. |
    | `Response.FinishReason` | Normalized to one of the constants in [`llm/legacy.go`](llm/legacy.go): `FinishReasonStop`, `FinishReasonLength`, `FinishReasonContentFilter`, `FinishReasonToolCalls`, `FinishReasonFunctionCall`, `FinishReasonUnknown`. |
    | Errors | Wrapped per §5 below. SDK error preserved in `Wrapped` so callers can `errors.Unwrap` for provider detail. |

    Each of OpenAI / Anthropic / Ollama implements this contract; their
    `map.go` files are the canonical examples for each row of the table.

    ## 4. Constructor Pattern (D-02 canonical)

    Every Phase-1 adapter uses functional options. Required signature:

    ```go
    func New(opts ...Option) (*X, error)
    ```

    Required options:

    | Option | Purpose | Notes |
    |--------|---------|-------|
    | `WithModel(string)` | Bind the model | **REQUIRED** — `New` returns error if empty |
    | `WithAPIKey(string)` | Override env-var fallback | Skipped for keyless providers (Ollama) |
    | `WithBaseURL(string)` | Custom endpoint | For httptest, Azure, proxies |
    | `WithHTTPClient(*http.Client)` | Custom transport | Retry middleware, OTel wrap |
    | `WithTimeout(time.Duration)` | Request timeout | Separate from `ctx` |

    Provider-specific extras are allowed. Use a provider-prefixed name where
    ambiguity exists (e.g., `openai.WithOrganization`, `anthropic.WithBetaHeader`,
    `ollama.WithHost`).

    Env-var fallback (when `WithAPIKey` is not given):
    - OpenAI: `OPENAI_API_KEY`
    - Anthropic: `ANTHROPIC_API_KEY`
    - Ollama: keyless; `OLLAMA_HOST` controls base URL

    Canonical sketch (from `llm-agent-providers/openai/options.go`):

    ```go
    type Option func(*config)

    func WithModel(m string) Option   { return func(c *config) { c.model = m } }
    func WithAPIKey(k string) Option  { return func(c *config) { c.apiKey = k } }
    func WithBaseURL(u string) Option { return func(c *config) { c.baseURL = u } }

    func New(opts ...Option) (*OpenAI, error) {
        cfg := config{}
        for _, opt := range opts { opt(&cfg) }
        if cfg.model == "" {
            return nil, errors.New("openai: WithModel is required")
        }
        if cfg.apiKey == "" {
            cfg.apiKey = os.Getenv("OPENAI_API_KEY")
        }
        // ... build SDK client; build Info{Provider, Model, Capabilities}
        return &OpenAI{...}, nil
    }
    ```

    Why functional options: Go-idiomatic (slog, gRPC, openai-go/v3 itself).
    Default values stay clean. Adding new options is BC-additive.

    ## 5. Error Taxonomy (D-03 canonical)

    Every adapter wraps SDK errors into one of `llm.AuthError`,
    `llm.RateLimitError`, `llm.InvalidRequestError`, or `llm.TransientError`,
    preserving the SDK error in the `Wrapped` field so consumers can
    `errors.As(err, &sdkErr)` for provider detail.

    **Recommended HTTP-status → typed-error mapping table:**

    | HTTP / Cause | Typed error |
    |--------------|-------------|
    | 401, 403 | `*llm.AuthError` |
    | 429 | `*llm.RateLimitError` |
    | 4xx other (400, 404, 422, etc.) | `*llm.InvalidRequestError` |
    | 5xx | `*llm.TransientError` |
    | `errors.Is(err, context.DeadlineExceeded)` | `*llm.TransientError` |
    | `errors.Is(err, context.Canceled)` | propagate as-is (NOT a typed llm error) |
    | network error (DNS, TCP reset, etc.) | `*llm.TransientError` |

    Provider-specific overrides are allowed; encode them in your adapter's
    `errors.go`. **Do NOT push provider-specific logic into the core llm package.**
    Examples already in the umbrella:

    - **OpenAI** `insufficient_quota` (429 with specific code) → still
      `*llm.RateLimitError`, but with `Reason: "quota_exhausted"`.
    - **Anthropic** `overloaded_error` (529) → `*llm.RateLimitError`
      (semantically rate-limit-like, NOT transient).
    - **Anthropic** `invalid_request_error` (400 with specific JSON code) →
      `*llm.InvalidRequestError`.
    - **Ollama** model-not-pulled (404 with body matching "not found") →
      `*llm.InvalidRequestError` (operator must run `ollama pull`, NOT a retry).

    Canonical wrapErr sketch (from `llm-agent-providers/openai/errors.go`):

    ```go
    func wrapErr(err error) error {
        if err == nil { return nil }
        if errors.Is(err, context.Canceled) { return err }
        if errors.Is(err, context.DeadlineExceeded) {
            return &llm.TransientError{Provider: "openai", Wrapped: err}
        }
        var apiErr *openai.Error
        if errors.As(err, &apiErr) {
            switch apiErr.StatusCode {
            case 401, 403:
                return &llm.AuthError{Provider: "openai", Wrapped: err}
            case 429:
                reason := ""
                if apiErr.Type == "insufficient_quota" {
                    reason = "quota_exhausted"
                }
                return &llm.RateLimitError{
                    Provider:   "openai",
                    RetryAfter: apiErr.Headers().Get("Retry-After"),
                    Reason:     reason,
                    Wrapped:    err,
                }
            case 500, 502, 503, 504:
                return &llm.TransientError{Provider: "openai", Wrapped: err}
            default:
                if apiErr.StatusCode >= 400 && apiErr.StatusCode < 500 {
                    return &llm.InvalidRequestError{Provider: "openai", Wrapped: err}
                }
            }
        }
        var netErr net.Error
        if errors.As(err, &netErr) {
            return &llm.TransientError{Provider: "openai", Wrapped: err}
        }
        return err
    }
    ```

    For SDKs that do NOT publicly expose their error type (e.g., Ollama
    returns plain `error`), inject a `statusCapturingTransport` via
    `WithHTTPClient` that records the last HTTP response's StatusCode in an
    atomic int32, then read it in `wrapErr`. See `llm-agent-providers/ollama/options.go`
    for the pattern.

    **Security note:** the `Wrapped` field carries the SDK's original error
    string. Adapter authors are responsible for ensuring the SDK does NOT
    include credentials (Authorization headers, API keys) in the error message
    surface. The mainstream SDKs redact correctly; verify when adopting a new SDK.
