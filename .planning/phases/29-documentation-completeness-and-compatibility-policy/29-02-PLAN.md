<objective>
Sweep every `llm-agent-rag` package and fill every exported-symbol
documentation gap, so every exported type, function, method, `Err*` value,
and exported const has a doc comment that starts with the symbol name (Go
convention). RAG-API-03.

Comment-only — the exported surface was frozen by Phase 28; this slice
documents it, it does not change it.
</objective>

<context>
- `docs/api-audit-v1.0.md` (Phase 28) is the **checklist** — it enumerates
  every exported symbol of every package. Walk it package-by-package.
- `go doc <pkg>` lists each exported symbol with its current first comment
  line; a symbol shown with only its signature (no following prose) is a
  gap. `go doc <pkg>.<Symbol>` expands a type to show which of its
  exported methods/fields are undocumented.
- The Go convention: a doc comment starts with the symbol name —
  `// Foo does …` for `func Foo`, `// Bar is …` for `type Bar`. `go doc`
  and `pkg.go.dev` rely on the prefix.
- `rag/options.go` already documents `Options`/`GlobalOptions`/
  `DriftOptions` fields well — that is the quality bar to match.
- Package-level comments were added in 29-01 — do not re-touch those.
- See `29-RESEARCH.md` Decision 2. KS-7 (doc bar is release-blocking),
  KS-2 (freeze, not redesign).
</context>

<tasks>
1. Walk every package in `docs/api-audit-v1.0.md`'s inventory. For each,
   run `GOWORK=off GOCACHE=/tmp/go-build go doc ./<pkg>` and identify every
   exported symbol with no doc comment or a non-name-prefixed comment.
   Expand types with `go doc ./<pkg>.<Type>` to catch undocumented exported
   methods and struct fields.
2. For every gap, add a doc comment that:
   - starts with the exact symbol name;
   - states what the symbol is / does in 1-3 lines (longer for the central
     types — `rag.System`, `retrieve.Retriever`, the `store` capability
     interfaces, the `graph` seams);
   - for sentinel `Err*` values, states the condition the error signals;
   - for exported struct fields lacking a comment, adds a short trailing or
     leading comment (match the `rag/options.go` style).
3. Include `adapter/llmagent` (inspect `model.go`/`tool.go` directly —
   `go doc` under the build tag may not be available; document
   `ModelAdapter`, its `Inner` field, `Generate`, and `AsTool`).
4. Do **not** rename, re-sign, add, or remove any symbol. If documenting a
   symbol surfaces a genuine naming/design wart, record it as a `/v2` note
   in `29-02-SUMMARY.md` — do not fix it here (Phase 28 froze the surface).
5. **Freeze hygiene — make the whole repo `gofmt`-clean.** Slice 29-01
   surfaced that `llm-agent-rag` was never fully `gofmt`-clean: ~10 files
   carry pre-existing trivial non-compliance (comment-column / struct-field
   alignment, import ordering within a group, one trailing-whitespace
   line) dating to `v0.6.0`. A v1.0 *stable* release must ship
   `gofmt`-clean. Run `gofmt -w .` across the repository so every file —
   the 29-02 doc edits and the pre-existing debt alike — is compliant.
   This is whitespace-only and zero-risk: it changes no token, no symbol,
   no behavior. Confirm with `git diff` that every `gofmt -w` hunk is pure
   formatting (no code change).
6. Run every command in `<verify>`.

Out of scope:
- Package-level comments — done in 29-01.
- `docs/compatibility.md`, the README — slice 29-03.
- Any change to a symbol's name or signature.
- Any non-whitespace edit to the files touched solely by the `gofmt -w`
  freeze-hygiene pass.
</tasks>
