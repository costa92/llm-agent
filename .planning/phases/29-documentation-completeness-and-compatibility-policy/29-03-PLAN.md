<tasks>
1. Create `docs/compatibility.md`. Cover, each as its own section:
   - **Import compatibility** — within `v1.x`, the API is additive-only:
     no exported symbol removed or renamed, no function/method signature
     changed, no exported struct field removed, no exported constant value
     changed that callers depend on. Permitted: new functions, new struct
     *fields*, new packages. **Call out the interface-method gotcha
     explicitly** — adding a method to an exported interface breaks every
     external implementer and is therefore a breaking change.
   - **Semantic versioning** — `v1.MINOR.PATCH`: PATCH = fixes, MINOR =
     backward-compatible additions, MAJOR = breaking. Go's module system
     enforces it.
   - **Breaking changes and `/v2`** — a breaking change requires a new
     import path `github.com/costa92/llm-agent-rag/v2`; there is no other
     mechanism.
   - **The `contract` sub-contract** — `contract/contract_test.go` pins the
     exact surface the core `llm-agent/rag` facade consumes; it changes
     only via coordinated PRs with `llm-agent`. Link
     `docs/core-compatibility.md`.
   - **External dependencies (`postgres`)** — `pgx`/`pgvector-go` are the
     only non-stdlib deps, isolated in the `postgres` package; state the
     minimum-version policy and that their major bumps surface via `go.mod`
     (they are *their* semver, not absorbed silently).
   - **`adapter/llmagent` coverage** — the build-tagged adapter is covered
     by the promise too; a build tag does not exempt a package from semver.
   - **`go.sum`** — committed, and correct for this module (it has the
     postgres-island deps); note this intentionally differs from the
     stdlib-only core `llm-agent`.
   - **Minimum Go version** — `go 1.26` is the v1.0 floor; minor-Go
     bumps are permitted within `v1.x`.
   - **Deprecation procedure** — a symbol is marked `// Deprecated: …` in a
     `v1.x` minor and removed only in a future `/v2`.
2. `README.md` — change the status line (~48) from "production-ready core,
   evolving ecosystem" to a stable-v1.0 status, with a link to
   `docs/compatibility.md`. Do not change any other part of the README.
