<objective>
Add a global-search evaluation harness to `eval` — run whole-corpus
questions through `AskGlobal` and score each answer with the RAG-Triad
`Judge` (groundedness against consulted community reports + answer
relevance). RAG-GRAPH3-06.
</objective>

<context>
- `rag.System.AskGlobal(ctx, q, GlobalOptions) (Answer, error)` (Phase 24).
  `Answer.Diagnostics.Global` carries `CommunityIDs`, `MapScores`, call
  counts.
- `eval` — `Dataset`/`Example`, `LLMJudge` →
  `Judgement{Groundedness, AnswerRelevance}`, `JudgeRequest`,
  `TriadEvaluator` (the `Asker`/`Judge` pattern to mirror). `RunGraphAB`
  measures chunk recall@k — the WRONG metric for global search.
- See `25-RESEARCH.md` Decision 4. Global search has no gold chunk set;
  eval is the Triad/Judge generation-side path only.
</context>

<tasks>
1. `rag/system.go` — add a `ConsultedReports []graph.CommunityReport` field
   to `GlobalDiagnostics`. `rag/global.go` — `AskGlobal` populates it with
   the community reports it actually consulted (the lazily-loaded /
   generated reports for the selected communities), so an evaluator can
   read the answer's grounding context off the `Answer` without store
   plumbing.
2. Create `eval/global.go` (package `eval`):
   - `GlobalAsker` interface:
     `AskGlobal(ctx, question string, opts rag.GlobalOptions) (rag.Answer, error)`.
   - `GlobalEvalResult struct` — `MeanGroundedness`, `MeanAnswerRelevance`,
     `Examples int`, and a `PerExample` detail slice (question, answer,
     judgement, consulted community IDs).
   - `GlobalEvaluator struct { Asker GlobalAsker; Judge Judge;
     MaxCommunities int }` with
     `Run(ctx, dataset Dataset) (GlobalEvalResult, error)`:
     - error if `Asker` or `Judge` is nil.
     - per `Example`: `AskGlobal` with `GlobalOptions{Namespace:
       ex.Namespace, MaxCommunities: …}`; build the judge context by
       concatenating `Answer.Diagnostics.Global.ConsultedReports` summaries;
       `Judge.Judge` for groundedness + answer-relevance; accumulate means.
     - the gold chunk/doc fields of `Example` are unused — global search
       has no chunk-recall notion; document that.
3. Tests `eval/global_test.go` — a CI gate: a scripted `generate.Model`
   (drives the summarizer + map/reduce inside `AskGlobal`) and a scripted
   `Judge`; build an in-memory store + graph + communities; assert
   `GlobalEvaluator.Run` scores every example, the means are computed, the
   judge receives the consulted-report context, and a nil-Asker/nil-Judge
   errors. Mirror the `eval` package's existing gate-test style.
4. Run every command in `<verify>`.

Out of scope:
- `docs/graphrag.md` — 25-03.
- Adding a new judge dimension (comprehensiveness) — v0.8 reuses
  `LLMJudge`'s groundedness + answer-relevance.
</tasks>
