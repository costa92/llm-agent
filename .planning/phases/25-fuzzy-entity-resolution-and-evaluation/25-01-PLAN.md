<objective>
Add embedding-similarity fuzzy entity resolution: an `EntityResolver` seam,
a `NoopEntityResolver` default, an `EmbeddingEntityResolver`, wired as an
opt-in pre-pass before `graph.Canonicalize` in `Import`. RAG-GRAPH3-05.
</objective>

<context>
- `graph.Canonicalize(entities, relations) Graph` — exact-match
  `(NormalizeName, type)` merge; resolves relation endpoint *names* to
  entity IDs and DROPS a relation whose endpoint matches no entity. The
  resolver must therefore rewrite relation endpoints too (see
  `25-RESEARCH.md` Decision 1).
- `embed.Embedder` — `Embed(ctx, text) (Vector, error)`, `Dimension()`;
  `embed.CosineSimilarity(a, b embed.Vector) float64` in `embed/hash.go`.
  `embed` is a stdlib-only leaf package — `graph` importing it adds no cycle.
- `rag.Import` (`rag/import.go`) accumulates `graphEnts`/`graphRels` then
  calls `graph.Canonicalize(graphEnts, graphRels)`. `Options.EntityExtractor`
  / `Options.CommunityDetector` show the option-wiring pattern.
- See `25-RESEARCH.md` Decisions 1-3. Keystones KG3-6 (determinism), KG3-8
  (opt-in pre-pass, conservative).
</context>

<tasks>
1. Create `graph/resolve.go` (package `graph`):
   - `EntityResolver` seam:
     `Resolve(ctx, entities []Entity, relations []Relation) ([]Entity, []Relation, error)`.
   - `NoopEntityResolver struct{}` — returns its inputs unchanged.
   - `EmbeddingEntityResolver struct { Embedder embed.Embedder; Threshold float64 }`:
     - `Threshold <= 0` → a conservative default (e.g. 0.92).
     - embed each entity's `Name` via `Embedder`; cluster entities of the
       **same `Type`** by cosine similarity `>= Threshold` (single-link,
       deterministic — scan entity pairs in sorted-by-`Name` order).
     - per cluster, pick the canonical `Name` deterministically: the longest
       name, tie-broken lexically lowest; rewrite every member entity's
       `Name` to it, and every relation whose `Source`/`Target` equals a
       member's original name to the canonical name.
     - a nil `Embedder` → a sentinel error
       (`ErrEntityResolverEmbedderRequired`).
     - fully deterministic: sorted iteration, fixed clustering order, no
       randomness.
2. `graph/graph.go` — update the package doc comment to name the `embed`
   seam alongside `generate` (the package now imports `embed`).
3. `rag/options.go` — add `Options.EntityResolver graph.EntityResolver`.
   `rag/system.go` — carry `s.entityResolver`; default to
   `graph.NoopEntityResolver{}` in `New` when unset.
4. `rag/import.go` — before `graph.Canonicalize(graphEnts, graphRels)`:
   `graphEnts, graphRels, err = s.entityResolver.Resolve(ctx, graphEnts,
   graphRels)` (wrap an error as `rag: resolve entities: %w`). With the
   `Noop` default, behavior is byte-identical to before.
5. Tests `graph/resolve_test.go` — `EmbeddingEntityResolver` against a
   **scripted embedder** (fixed vectors): "Acme"/"Acme Corp" of the same
   type collapse to one canonical name and a relation endpoint following
   suit; entities of different types do NOT merge even at high similarity;
   unrelated entities stay distinct; the canonical-form pick is
   deterministic; `NoopEntityResolver` is identity; nil embedder errors.
6. Tests `rag/resolve_test.go` — an `Import` with an `EmbeddingEntityResolver`
   in `Options` produces a graph where the near-duplicate entities are a
   single canonical node with unioned provenance and the relation survives;
   the default (no resolver) import is unchanged.
7. Run every command in `<verify>`.

Out of scope:

- `docs/graphrag.md` — 25-03.
</tasks>
