```yaml
# .github/workflows/umbrella.yml
# INFRA-05: builds all 4 repos against this PR's llm-agent on every PR to main.
# Sister repos must exist on GitHub for the cross-repo checkouts to succeed
# (Phase 0 plan 00-03 creates them).
name: umbrella

on:
  pull_request:
    branches: [main]
  workflow_dispatch:

concurrency:
  group: umbrella-${{ github.ref }}
  cancel-in-progress: true

jobs:
  cross-repo-build:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - name: Checkout llm-agent (this PR)
        uses: actions/checkout@v4
        with:
          path: llm-agent

      - name: Checkout llm-agent-providers (main)
        uses: actions/checkout@v4
        with:
          repository: costa92/llm-agent-providers
          path: llm-agent-providers
          ref: main

      - name: Checkout llm-agent-otel (main)
        uses: actions/checkout@v4
        with:
          repository: costa92/llm-agent-otel
          path: llm-agent-otel
          ref: main

      - name: Checkout llm-agent-customer-support (main)
        uses: actions/checkout@v4
        with:
          repository: costa92/llm-agent-customer-support
          path: llm-agent-customer-support
          ref: main

      - uses: actions/setup-go@v5
        with:
          go-version-file: llm-agent/go.mod
          cache: false

      - name: Verify scripts/workspace.sh is byte-identical across all 4 checkouts (INFRA-03)
        run: |
          set -euo pipefail
          # INFRA-03: scripts/workspace.sh MUST be byte-identical in every repo.
          # Compute the SHA-256 of the core repo's copy and assert each sister
          # repo's copy matches. Any drift fails the umbrella job loudly.
          core_sha=$(sha256sum llm-agent/scripts/workspace.sh | awk '{print $1}')
          echo "core scripts/workspace.sh sha256: $core_sha"
          for repo in llm-agent-providers llm-agent-otel llm-agent-customer-support; do
            sister_sha=$(sha256sum "$repo/scripts/workspace.sh" | awk '{print $1}')
            if [ "$sister_sha" != "$core_sha" ]; then
              echo "::error::$repo/scripts/workspace.sh sha256 ($sister_sha) does not match core ($core_sha)"
              echo "Diff against core:"
              diff -u "llm-agent/scripts/workspace.sh" "$repo/scripts/workspace.sh" || true
              exit 1
            fi
            echo "$repo scripts/workspace.sh sha256 matches core: ✓"
          done

      - name: Initialize go.work pointing all 4 modules at this PR's llm-agent
        run: |
          go work init ./llm-agent ./llm-agent-providers ./llm-agent-otel ./llm-agent-customer-support
          # All 4 modules now resolve github.com/costa92/llm-agent to ./llm-agent
          # (this PR's HEAD), regardless of the version pinned in their go.mod files.
          go work sync || true   # best-effort sync; failures are not fatal because sister repos may have no source files yet (Phase 0 skeleton)

      - name: Build llm-agent (this PR)
        run: |
          cd llm-agent
          go vet ./...
          go build ./...
          go test ./... -count=1

      - name: Build llm-agent-providers against this PR's llm-agent
        run: |
          cd llm-agent-providers
          go vet ./...
          go build ./...
          go test ./...

      - name: Build llm-agent-otel against this PR's llm-agent
        run: |
          cd llm-agent-otel
          go vet ./...
          go build ./...
          go test ./...

      - name: Build llm-agent-customer-support against this PR's llm-agent
        run: |
          cd llm-agent-customer-support
          go vet ./...
          go build ./...
          go test ./...
```

Notes:
- The `actions/checkout@v4` for the local PR repo deliberately uses `path: llm-agent` so all 4 module clones live as siblings under the runner's workspace root, matching the layout `scripts/workspace.sh` writes locally.
- `go-version-file: llm-agent/go.mod` resolves to the version pinned in the PR's go.mod — same Go version as the per-repo `test.yml` uses.
- `cache: false` for `setup-go` because the 4-module workspace has 4 separate go.sum surfaces and the Action's caching key is single-module-aware. Disabling caching is acceptable here — umbrella runs only on PR (not push to main), so the cost is bounded.
- `go work sync || true` after `go work init` is a best-effort sync; in Phase 0 the sister repos have no Go source, so `sync` cannot fully resolve `require` lines. The `|| true` swallows non-fatal sync errors. This is fine: the per-repo `go vet/build/test` calls below are the actual assertions.
- The umbrella workflow is INTENTIONALLY decoupled from the per-repo `test.yml` (both fire on the same PR but run as separate jobs in different workflows). They check different invariants: per-repo CI checks the repo in isolation; umbrella CI checks the cross-repo composition.

```yaml
# .github/workflows/release-precheck.yml
# INFRA-04: rejects `replace` directives on tagged-release branches.
# Lifted from PITFALLS.md Pitfall 12 §"How to avoid". Identical across all 4 repos.
name: release-precheck

on:
  push:
    branches:
      - 'release/**'
  pull_request:
    branches:
      - 'release/**'

jobs:
  no-replace:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - name: Reject `replace` directives in go.mod
        run: |
          replace_count=$(go mod edit -json | python3 -c '
          import json, sys
          data = json.load(sys.stdin)
          replaces = data.get("Replace") or []
          print(len(replaces))
          ')
          if [ "$replace_count" -ne 0 ]; then
            echo "::error::go.mod contains $replace_count replace directive(s) — these MUST be removed before tagging."
            echo "Detail:"
            go mod edit -json | python3 -m json.tool | grep -A 100 '"Replace"' || true
            exit 1
          fi
          echo "OK: go.mod has zero replace directives."
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Umbrella workflow trigger | `pull_request: branches: [main]` runs in PR-fork context with default `GITHUB_TOKEN` (limited permissions). NOT `pull_request_target` — the privilege-escalation vector for malicious forks. Verified by Task 1's `! grep -q 'pull_request_target'` assertion. |
| Cross-repo checkout | `actions/checkout@v4` with `repository: costa92/<sister>` works for public repos without a token. The runner's permissions are read-only on the sister repos — cannot push, cannot modify settings. |
| Release-precheck `release/**` trigger | Both `push` and `pull_request` triggers on `release/**` branches. The branch pattern is the agreed gate point — releases happen via release branches, NOT directly via tag pushes. RESEARCH.md A3 acknowledges this is an assumed convention that may need extending to `tags:` triggers if the release process changes. |
| `go mod edit -json` parsing | Reads only the local `go.mod` from the runner's workspace. No network calls, no third-party data. Python JSON parsing is stdlib-only. |
