<objective>
Ship the compose-with-everything integration test for the `policy`
package. Per Decision G in 36-RESEARCH.md, this test does NOT import
`llm-agent-otel/otelmodel` — that would violate the stdlib-only rule.
Instead, the test ships a tiny in-test `observerModel` (~20 LOC) that
mirrors `otelmodel.Wrap`'s capability-preserving shape: a struct that
satisfies `llm.ChatModel` + the 3 capability interfaces via type-switch,
with atomic counters for `Generate` / `Stream` / `Next` invocations.

The integration test proves five things: (1) capability assertions
survive both wrappers (`policy.Wrap(observerModel.Wrap(scriptedLLM))`
preserves `ToolCaller` / `Embedder` / `StructuredOutputs`); (2) a Block
decision on `PreGenerate` short-circuits BEFORE the observer's
`Generate` is reached (proves outer-denies-first per KC-3); (3)
streaming events flow through both layers correctly (observer counts
deltas, policy fires per-event gates); (4) budget exhaustion at the
`generateFromPrompt` chokepoint fires BEFORE the policy decorator
(proves budget chokepoint sits underneath the wrapper stack); (5) all
5 agent paradigms propagate `BlockedError` correctly through `Agent.Run`.
Pure stdlib + the 36-01/02 surfaces + the existing `budget` and
`agents` packages from Phase 35. CC-2.
</objective>

- **TestCompose_BudgetBeatsPolicyAtChokepoint design.** From Phase 35's
  35-RESEARCH.md §"Carry-forward notes": the v1.2 stack is
  `policy.Wrap(otelmodel.Wrap(provider))` with budget enforced at
  `generateFromPrompt` UNDERNEATH all wrappers. Test reproduces this
  invariant: wrap a ScriptedLLM in `policy.Wrap(...)` with a gate that
  WOULD block, attach `budget.Budget{MaxCalls: 0}` (impossible —
  zero is "no cap"; use `MaxCalls: -1` or charge upstream to deplete;
  actually use `budget.WithBudget(ctx, Budget{MaxCalls: 1})` and
  pre-charge via a previous call so the next is denied — or use
  `MaxTokens: 1` so the first call charges 1, then the second-call
  pre-check returns ErrCallsExceeded). Simpler approach: use
  `budget.Budget{MaxCalls: 0}` is "no cap" (0 means infinity per
  budget package), so we need a non-zero cap. Use `MaxCalls: 1`, run
  one call to deplete, then assert the SECOND call returns
  `budget.ErrCallsExceeded` not `policy.ErrBlocked` even though the
  policy gate would also block. Run via `agents.SimpleAgent` so the
  chokepoint actually fires (the chokepoint is in
  `agent_chatmodel.go::generateFromPrompt` which paradigms invoke,
  NOT in direct `model.Generate` calls — verify by reading
  `simple.go:52`).
