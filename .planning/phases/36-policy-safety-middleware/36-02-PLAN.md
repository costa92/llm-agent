  **stateless**. The 3 built-ins hold immutable config (rule slices,
  byte caps) at construct time but accumulate no per-call state. The
  decorator's `streamReader` from 36-01 handles per-stream state
  (`started`, `closed` flags); gates do NOT keep per-stream buffers.
  This makes concurrent `Generate` calls race-clean (one gate value can
  be shared by many goroutines).

  `policy/pii.go` top comment.
