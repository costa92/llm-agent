<objective>
Add community summarization: a `CommunityReport` type, a `CommunitySummarizer`
seam, an `LLMCommunitySummarizer`, a deterministic `CommunityContentHash`,
and report persistence behind the `store.CommunityStore` capability
(in-memory + postgres). RAG-GRAPH3-03.
</objective>

<context>
- `graph.Community{ID, Level, ParentID, EntityIDs, RelationIDs}` (Phase 23).
  `graph` already imports `generate` — see `graph/extract.go`'s
  `LLMEntityExtractor` for the LLM-backed-type + lenient-parse precedent.
- `generate.Model.Generate(ctx, Request) (Response, error)`;
  `Request{SystemPrompt, Messages, Metadata}`, `Response{Text, Usage}`.

- See `24-RESEARCH.md` Decisions 1 and 2.
</context>

<tasks>
1. Create `graph/summary.go` (package `graph`):
   - `CommunityReport struct { CommunityID, Title, Summary, ContentHash string }`.
   - `CommunityContentHash(c Community) string` — `crypto/sha256` over the
     sorted `EntityIDs` then `RelationIDs` (a fixed separator); hex digest.
     Deterministic.
   - `CommunitySummarizer` seam:
     `Summarize(ctx, c Community, g Graph) (CommunityReport, error)`.
   - `LLMCommunitySummarizer struct { Model generate.Model }` — builds a
     prompt from the community's member entity names+descriptions and
     relation descriptions (look them up in `g` by ID), asks for a short
     title + a paragraph summary; lenient parse (a `Title:` line + the rest
     as `Summary`, tolerating fences/preamble — mirror `LLMEntityExtractor`).
     Sets `CommunityID` and `ContentHash = CommunityContentHash(c)`. A nil
     `Model` returns a sentinel error (`ErrCommunitySummarizerModelRequired`).
2. Tests `graph/summary_test.go` — `LLMCommunitySummarizer` against a
   scripted model: a canned report parses; malformed output (no title line,
   junk) parses leniently without error; `ContentHash` is set and stable;
   `CommunityContentHash` is deterministic and changes when membership
   changes.
3. Extend `store.CommunityStore` in `store/store.go` with:
   ```go
   PutCommunityReport(ctx context.Context, namespace string, report graph.CommunityReport) error
   CommunityReport(ctx context.Context, namespace, communityID string) (graph.CommunityReport, bool, error)
   ```
4. `store/community.go` — implement both on `*InMemoryStore` (a
   `reports map[string]map[string]graph.CommunityReport` field —
   namespace → communityID → report — initialized in `NewInMemoryStore`).
   `CommunityReport` returns `found=false` for an unknown id, no error.
5. `postgres/community.go` + `postgres/postgres.go` — implement both on
   `*Store`; add a `<table>_community_reports` table in `Migrate()`
   (idempotent, `isSafeIdent`-guarded; columns
   `namespace, community_id, title, summary, content_hash`, PK
   `(namespace, community_id)`). `PutCommunityReport` upserts (`ON CONFLICT`).
6. Extend `storetest.RunCommunityConformance` with a report round-trip:
   `PutCommunityReport` then `CommunityReport` returns it unchanged; an
   unknown id is `found=false`/no-error; a second `Put` for the same id
   overwrites; namespaces isolated. Update `store/community_test.go` and the
   postgres conformance test as needed (both already call the suite).
7. Run every command in `<verify>`.

Out of scope:
- `AskGlobal` / global search — 24-02.
- `PrewarmCommunityReports`, the worked example — 24-03.
</tasks>
