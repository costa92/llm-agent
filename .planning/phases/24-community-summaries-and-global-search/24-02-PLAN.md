<objective>
Add `rag.System.AskGlobal` — a map-reduce global-search answer path over
community reports, distinct from `Ask`. Community selection, lazy report
generation, per-community map, score-ranked reduce. RAG-GRAPH3-04.
</objective>

<context>
- `graph.CommunitySummarizer` / `LLMCommunitySummarizer` /
  `CommunityContentHash` and `store.CommunityStore`'s
  `Communities`/`CommunityReport`/`PutCommunityReport` come from 24-01.
- `generate.Model.Generate(ctx, Request) (Response, error)`. `rag.System`
  carries `s.model generate.Model`; `Ask` installs a fresh `obs.Counter`
  via `obs.WithCounter(ctx, counter)` and times stages — `AskGlobal`
  mirrors that shell.
- `Answer{Text, Hits, Prompt, Citations, Diagnostics, Trace}`. Reuse it.
- `AskGlobal` is a SEPARATE path (KG3-4): no `s.retrieve`, no reranker, no
  packer. See `24-RESEARCH.md` Decision 3.
</context>

<tasks>
1. `rag/options.go` — add `Options.CommunitySummarizer graph.CommunitySummarizer`
   and a `GlobalOptions struct { Namespace string; MaxCommunities int }`
   (`MaxCommunities <= 0` → a sane default, e.g. 8).
2. `rag/system.go` — carry `s.communitySummarizer` (wired in `New`); add a
   `GlobalDiagnostics` struct (`CommunityIDs []string`, `MapScores
   map[string]int`, `MapCalls int`, `ReduceCalls int`) and a
   `Diagnostics.Global GlobalDiagnostics` field (additive).
3. Create `rag/global.go` (package `rag`):
   - `func (s *System) AskGlobal(ctx, question string, opts GlobalOptions)
     (Answer, error)`:
     - `s.model == nil` → `ErrModelRequired`. Install a fresh `obs.Counter`.
     - type-assert `s.store` for `store.CommunityStore`; if absent → empty
       `Answer`, no error (graceful).
     - `Communities(ns)`; if none → empty `Answer`, no error. Select the
       **coarsest** level (max `Level`). If the count exceeds
       `MaxCommunities`, rank by query-token overlap with member entity
       names (lowercase whitespace tokens; deterministic, tie-break by
       community ID) and cap.
     - **lazy report** per selected community: `CommunityReport(ns, id)`;
       use it iff found AND `report.ContentHash == graph.CommunityContentHash(c)`;
       otherwise require `s.communitySummarizer` (nil → `ErrCommunitySummarizerRequired`),
       `Summarize`, then `PutCommunityReport`.
     - **map**: for each report, `s.model.Generate` with a prompt carrying
       the report title+summary and the question; parse a helpfulness score
       (0-100, lenient — default 0 if unparseable) and the partial-answer
       text.
     - **reduce**: drop score-0 partials; sort by score desc (tie-break by
       community ID); `s.model.Generate` once to synthesize the survivors
       into the final answer. No survivor → a graceful "no relevant
       community information" answer text.
     - return `Answer{Text: final, Diagnostics: Diagnostics{Global: …,
       Metrics: …}}` — record selected community IDs, per-community map
       scores, and map/reduce call counts.
4. Tests `rag/global_test.go` — a scripted `generate.Model` whose responses
   are keyed so map and reduce are distinguishable; build an in-memory store
   with a small graph + detected communities (`graph.LouvainDetector`):
   - `AskGlobal` selects coarsest-level communities, lazily summarizes them
     (cache miss → summarizer called; second call → cache hit, no
     re-summarize), maps, reduces; the final `Answer.Text` is the reduce
     output and `Diagnostics.Global` is populated.
   - a stale cached report (`ContentHash` mismatch) triggers re-summarize.
   - `AskGlobal` over a non-`CommunityStore` store, and over a namespace
     with no communities → empty `Answer`, no error.
   - `MaxCommunities` caps the selected set.
5. Run every command in `<verify>`.

Out of scope:
- `PrewarmCommunityReports`, local enrichment, the worked example — 24-03.
- Hierarchy-level selection knobs — v0.8 fixes on the coarsest level.
</tasks>
