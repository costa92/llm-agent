<objective>
RAG-OPS-01 demands "conformance coverage against the core store
contract" so any new backend can prove correctness against the same
bar as the in-memory baseline. Today each store ships its own ad-hoc
test set, and the postgres store has only a smoke check.

This slice introduces a shared conformance suite under
`store/storetest/` and wires both the in-memory store (always-on) and
the postgres store (env-gated) into it. Future backends — Qdrant,
SQLite-vec, etc. — opt in with a single call.

The tracing-hooks half of the original 12-02 scope is deferred to
12-03 so this slice stays focused.
</objective>

<context>
- The store contract is small (7 methods) but has subtle invariants:
  namespace isolation, security-filter override, metadata subset match,
  ErrNotFound on missing rows, ErrDimensionMismatch on vector-dim
  mismatch.
- Existing in-memory tests (`store/inmemory_test.go`) cover most of
  these as bespoke cases; they should keep working unchanged. The new
  conformance test is **additive** — proving the same store passes both
  the suite and its existing bespoke tests is the bar.
- Postgres store (`postgres/postgres.go` from 12-01) needs unique
  state per subtest to avoid cross-test pollution. Plan: each subtest
  gets a fresh table whose name encodes `t.Name()`, registered via
  `t.Cleanup` to DROP after.
- The conformance subpackage convention follows stdlib (httptest,
  fstest, iotest) — `store/storetest` lives next to `store/` and
  imports `testing`, so it cannot be in `store/` itself.
- Backend capability matrix is small in v0.5; we mark capabilities via
  optional `storetest.Option` values rather than requiring backends to
  implement a separate Capability interface.
</context>

<tasks>
1. New package `store/storetest/`:
   - `storetest.go` exports `Factory`, `Option`, `RunConformance`
     ```go
     type Factory func(t *testing.T) store.Store

     type Option func(*config)
     func WithDimensionStrict() Option
     // future: WithoutSecurityFilters, WithMaxNamespace, ...

     func RunConformance(t *testing.T, factory Factory, opts ...Option)
     ```
   - subtests (all use `t.Run(name, ...)`):
     - `Upsert and Get round-trip` — upsert one chunk with all fields
       populated, `Get` it, deep-equal expectations
     - `Search returns nearest first` — three chunks with distinct
       vectors, query nearest, assert ranking
     - `Search respects namespace` — two namespaces with same query
       vector; Search scoped to one returns only that namespace's hits
     - `Filter narrows results` — metadata filter on a key/value
       subset; non-matching chunks are excluded
     - `Security filters override caller filters` — a caller filter
       that would widen the set is overridden by a stricter
       security filter
     - `List returns namespace chunks` — same expectations as List
       semantics
     - `Get on missing returns ErrNotFound` — `errors.Is(...,
       store.ErrNotFound)`
     - `Remove returns ErrNotFound for missing` — same check
     - `Remove deletes` — round-trip Upsert → Remove → Get returns
       ErrNotFound
     - `RemoveByFilter returns count and removes` — assert returned
       count matches actual removals
     - `Stats reports namespace count and dim` — fresh store
       Upsert N chunks; Stats(namespace) returns N and dim
     - `Dimension mismatch returns error` — opt-in via
       `WithDimensionStrict()`; Upsert with wrong-dim vector returns
       `store.ErrDimensionMismatch`
2. Wire the in-memory store into the suite:
   - new file `store/inmemory_conformance_test.go`:
     ```go
     func TestInMemoryStoreConformance(t *testing.T) {
         storetest.RunConformance(t, func(t *testing.T) store.Store {
             return store.NewInMemoryStore(2)
         }, storetest.WithDimensionStrict())
     }
     ```
   - Existing `store/inmemory_test.go` stays as-is. The conformance
     test is additive; no bespoke tests are deleted.
3. Wire the postgres store into the suite, env-gated:
   - new file `postgres/postgres_conformance_test.go`:
     ```go
     func TestPostgresStoreConformance(t *testing.T) {
         dsn := os.Getenv(liveEnvVar)
         if dsn == "" { t.Skip(...) }
         // create one pool for the whole suite
         storetest.RunConformance(t, func(t *testing.T) store.Store {
             // table name derived from t.Name() via a sanitizer
             // (lowercase, replace / with _, append a random suffix)
             // build Store, call Migrate, register t.Cleanup to DROP
         }, storetest.WithDimensionStrict())
     }
     ```
4. The smoke test from 12-01 (`TestPostgresStore_LiveSmoke` /
   `TestPostgresStore_LiveMetadataFilter`) stays — it's now a thin
   end-to-end check on top of the broader conformance coverage. Do
   not delete it; it documents the AfterConnect / RegisterTypes
   pattern in one short test that's easy to read.
5. Verify per `<verify>`. The in-memory conformance run is the
   primary green signal; the postgres conformance run only runs when
   the env var is set.

Out of scope:
- Tracing hooks for Import / Retrieve / Pack / Ask → 12-03
- Capability matrix as a typed interface → not needed for v0.5
- Performance benchmarks → separate slice if/when needed
</tasks>
