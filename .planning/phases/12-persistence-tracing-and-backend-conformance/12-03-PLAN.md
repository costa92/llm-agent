<objective>
RAG-OPS-02 says "tracing and evaluation hooks exist for import,
retrieve, pack, and ask flows." Today `Retrieve` and `Ask` already
carry rich trace data inside their return values, but consumers have
no observation point — they have to inspect the returned struct.
`Import` has no trace at all. `Pack` is internal to `Ask` and its
data is already surfaced via `AskTrace`.

This slice adds an explicit `Observer` hook on `rag.Options`. The
hook fires after each top-level operation with the operation's full
trace. OTel adapters in `llm-agent-otel` (sister repo) wire to this
hook to emit spans + attributes without touching the rag facade
internals.

Design choice: callback-struct (optional `func` fields), not an
interface. Reasons:

- partial implementations are trivial (leave fields nil)
- no required noop base struct
- nil-check before call is one branch, zero allocation when unset
- still composable from OTel side via a single helper
</objective>

<context>
- `Retrieve` already returns `retrieve.Trace` from inside the rag
  facade as part of `Ask` and via the existing `(*System).retrieve`
  helper. The public `(*System).Retrieve` returns hits only — the
  trace is dropped. The new hook restores that signal externally.
- `Ask` builds a complete `rag.Trace` (a.k.a. AskTrace) before
  returning. The hook fires right before return.
- `Import` builds `ingest.ImportResult` (counts + chunk IDs) but
  loses other useful context: embed call count, replace-source
  bookkeeping, namespace. The new `rag.ImportTrace` adds those.
- Pack data already lives inside `AskTrace`
  (`PackedChunkIDs`, `DroppedChunkIDs`, `RerankedChunkIDs`,
  `SelectedChunkIDs`). An `OnPack` hook would duplicate that, so we
  intentionally do not add one. If a consumer needs Pack-level
  granularity, they pull it from `AskTrace`.
- Timing / span creation lives in the consumer (OTel adapter). This
  slice does not add `time.Now()` calls to the rag facade —
  attempting to measure here would predetermine the span shape and
  conflict with OTel's own context-propagated timing.
</context>

<tasks>
1. New file `rag/observer.go`:
   ```go
   // ImportTrace captures observable data from an Import call.
   type ImportTrace struct {
       Namespace     string
       Documents     int
       Chunks        int
       ChunkIDs      []string
       EmbedCount    int
       ReplaceSource bool
       RemovedChunks int
   }

   // Observer holds optional callbacks that fire after each top-level
   // rag operation completes successfully. Each callback is nil-safe;
   // unset callbacks are skipped without allocation.
   //
   // Errors returned by the underlying operation short-circuit before
   // the callback fires — observers see only successful runs.
   type Observer struct {
       OnImport   func(ctx context.Context, trace ImportTrace)
       OnRetrieve func(ctx context.Context, trace retrieve.Trace)
       OnAsk      func(ctx context.Context, trace Trace)
   }
   ```
2. Add `Observer Observer` field to `rag.Options` in `rag/options.go`.
3. `rag/system.go`: store the observer on `*System` during `New(...)`.
4. `rag/import.go`: build an `ImportTrace` as the loop runs (count
   embed calls and any removed chunks during ReplaceSource), then
   emit on success.
5. `rag/retrieve.go`: after `s.retrieve(...)` succeeds, call
   `s.observer.OnRetrieve(ctx, trace)` if set. Note: `(*System).Retrieve`
   currently discards the trace — that stays for the caller-facing
   return shape, but the observer now sees it.
6. `rag/ask.go`: after the `Answer` is fully built, but before
   `return`, call `s.observer.OnAsk(ctx, answer.Trace)` if set.
7. Tests in a new `rag/observer_test.go`:
   - `TestObserverFiresOnImport` — set `OnImport`, call `Import`,
     assert the callback ran exactly once with the right
     `Documents`, `Chunks`, `Namespace`
   - `TestObserverFiresOnRetrieve` — set `OnRetrieve`, call
     `Retrieve`, assert the callback ran with non-empty trace
   - `TestObserverFiresOnAsk` — set `OnAsk`, call `Ask`, assert the
     callback ran with the same trace the answer carries
   - `TestObserverNilCallbacksAreSafe` — Observer with all-nil
     callbacks: no panic on any operation
8. Verify per `<verify>`.

Out of scope:
- OTel-side wiring (lives in `llm-agent-otel`, not here)
- Per-subphase hooks (chunk-by-chunk Embed, per-route Retrieve
  sub-spans) — flat trace data only
- Error-path callbacks — observers fire only on success. Error
  observability is the consumer's responsibility via the returned
  error
</tasks>
