<objective>
Add a PostgreSQL + pgvector implementation of `store.Store` so the
standalone SDK can be deployed beyond in-memory demos. This is the
production-line backend referenced by RAG-OPS-01 and the natural fit
with the customer-support refsvc's existing PostgreSQL session-store
(Phase 6 Plan 05).

The scope is *the backend itself*, not the full conformance suite — that
lands in 12-02. This slice delivers:

1. a `postgres` package implementing `store.Store`
2. schema migration helper (chunks table + ivfflat vector index)
3. the first-ever non-stdlib deps in `llm-agent-rag` (pgx/v5 +
   pgvector-go)
4. a live-Postgres smoke test guarded by `LLM_AGENT_RAG_PG_URL` so CI
   without Postgres still passes
</objective>

<context>
- `llm-agent-rag` was zero-dep up to this point. Adding the pgvector
  driver is the planned moment to break that (the CORE repo
  `llm-agent` stays stdlib-only — that constraint does *not* apply to
  sister repos).
- Canonical pgvector Go path in 2026 is `github.com/jackc/pgx/v5` for
  the driver and `github.com/pgvector/pgvector-go` for the `Vector`
  type. `pgvector-go` integrates with pgx via a registered codec.
- The store contract (from `store/store.go`):
  ```go
  type Store interface {
      Upsert(ctx, []StoredChunk) error
      Search(ctx, Query) ([]Hit, error)
      List(ctx, namespace, filters, securityFilters) ([]StoredChunk, error)
      Get(ctx, id) (StoredChunk, error)
      Remove(ctx, id) error
      RemoveByFilter(ctx, namespace, filters) (int, error)
      Stats(ctx, namespace) (Stats, error)
  }
  ```
- `StoredChunk` carries vector, namespace, doc id, section id, section
  path, heading, title, content, metadata. Schema must store all of
  these losslessly.
- `Filter` is `map[string]any` — for v1 the postgres implementation
  matches the in-memory store's filter behavior (deep-equality on
  scalar values; complex filters can land later).
- Conformance suite scaffolding (a shared test harness all stores
  satisfy) is deferred to 12-02; this slice only adds a smoke test so
  the implementation has at least one live-Postgres assertion.
</context>

<tasks>
1. `go.mod`: add `github.com/jackc/pgx/v5` and
   `github.com/pgvector/pgvector-go` via `go get`. Note: `go.sum` will
   appear for the first time in this repo.
2. New package `postgres/`:
   - `postgres.go`:
     - `type Store struct { pool *pgxpool.Pool; table string }`
     - `type Config struct { Table string }` (default `chunks`)
     - `New(pool *pgxpool.Pool, cfg Config) *Store` — constructor
     - `Migrate(ctx) error` — `CREATE EXTENSION IF NOT EXISTS vector`,
       create table, create ivfflat index on `embedding`
     - schema (one table):
       ```sql
       CREATE TABLE IF NOT EXISTS {table} (
         id            TEXT PRIMARY KEY,
         namespace     TEXT NOT NULL,
         doc_id        TEXT NOT NULL,
         section_id    TEXT,
         section_path  TEXT[],
         heading       TEXT,
         title         TEXT,
         content       TEXT NOT NULL,
         metadata      JSONB,
         embedding     VECTOR(NULL) -- dimension declared at runtime via Migrate
       )
       ```
       Use `vector` type without a fixed dimension at table-creation
       time (allow per-namespace dims later); enforce dimension match
       on Upsert via Go-side check against the first row in the
       namespace.
     - `Upsert`: batch insert with `ON CONFLICT (id) DO UPDATE` on
       every chunk field including `embedding`. Validate vector
       dimension matches existing namespace rows; return
       `store.ErrDimensionMismatch` on mismatch.
     - `Search`: parameterized `ORDER BY embedding <=> $1 LIMIT $2`
       (cosine distance via pgvector). Apply filters / security
       filters as additional `WHERE` clauses on `metadata @> $N`.
     - `List`: select all matching `namespace + filters`.
     - `Get` / `Remove` / `RemoveByFilter` / `Stats`: straight SQL
       against the same table.
   - All exported methods return `store.ErrNotFound` /
     `store.ErrDimensionMismatch` where appropriate so callers can
     `errors.Is` on them.
3. `postgres/postgres_test.go`:
   - `TestPostgresStore_LiveSmoke`:
     - read `LLM_AGENT_RAG_PG_URL` env var; `t.Skip(...)` if unset
     - open a `pgxpool.Pool`, register the pgvector codec
     - construct `Store`, call `Migrate(ctx)`
     - upsert two chunks with 2-dim vectors `[1,0]` and `[0,1]`,
       namespace `test`
     - `Search` with query vector `[1,0]` → expect the first chunk
       ranked first
     - `Remove` the first chunk → `Get` returns
       `store.ErrNotFound`
     - cleanup: `DROP TABLE` the test table
   - Optional second test, also env-gated:
     `TestPostgresStore_LiveMetadataFilter` — upsert two chunks with
     different `metadata.lang`, assert filter narrows results.
4. **Do not** wire the postgres store into the rag facade in this
   slice — keep it opt-in by consumer code. (The
   `rag.New(Options{Store: postgres.New(...)})` path already works
   because `Options.Store` accepts any `store.Store`.)
5. Run the verify bundle in `<verify>`. The standalone repo will
   acquire its first `go.sum` — that is expected and intentional.

Out of scope for this slice:
- Conformance suite harness (`store.ConformanceSuite(t, factory)`) —
  lands in 12-02.
- Tracing hooks — lands in 12-02.
- Live-Postgres in CI — left as opt-in env var; CI wiring lands later.
- HNSW / quantization / advanced indexing tuning.
- pgvector version pinning beyond the minimum that supports the
  `vector` type.
</tasks>
