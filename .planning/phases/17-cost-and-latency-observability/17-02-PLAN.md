<objective>
17-01 added stage durations and call counts but left `obs.Metrics.Tokens`
zero. This slice completes RAG-OBS-01's token half: `generate.Response`
gains an optional `Usage` field, and the ask flow records token cost into
`obs.Metrics.Tokens` ‚Äî using the model's reported usage when present, and a
`pack.TokenCounter` estimate (flagged `Estimated`) when it is not.
</objective>

<context>
- `generate/types.go` ‚Äî `Response struct { Text string }`. Changing it is an
  **additive** public-API change: a new field, zero for every existing
  caller and for the bundled scripted models.
- `rag/ask.go` `Ask` ‚Äî calls `resp, err := s.model.Generate(ctx, req)`;
  17-01 already assembles an `obs.Metrics` here and assigns it to
  `Diagnostics.Metrics`. The rendered prompt is `req` (a
  `generate.Request{SystemPrompt, Messages}`).
- `pack` package ‚Äî `TokenCounter` interface (`Count(text) int`) and
  `SimpleCounter` (words√ó1.3 + CJK runes). Reusable, no new dependency.
- `obs.TokenUsage{PromptTokens, CompletionTokens, TotalTokens int; Estimated
  bool}` and `obs.Metrics.Tokens` exist from 17-01.
- See `17-RESEARCH.md` Decision 4.
</context>

<tasks>
1. In `generate/types.go` add:
   ```go
   // Usage is the token cost of one generation. Model adapters that know
   // real counts populate it; it is zero when usage is unknown.
   type Usage struct {
       PromptTokens     int
       CompletionTokens int
       TotalTokens      int
   }
   ```
   and add a `Usage Usage` field to `Response`.
2. In `rag/ask.go`, after `s.model.Generate` returns `resp`, derive the
   token usage for `obs.Metrics.Tokens`:
   - If `resp.Usage` is non-zero (any of the three counts > 0): copy it into
     `obs.TokenUsage` with `Estimated = false`. If `TotalTokens` is 0 but the
     two parts are set, fill `TotalTokens = Prompt + Completion`.
   - Else estimate with a `pack.TokenCounter` (`pack.SimpleCounter{}` unless
     the system already holds a configured counter ‚Äî use `SimpleCounter`):
     `PromptTokens` = count of the rendered prompt text (system prompt +
     all message contents joined), `CompletionTokens` = count of `resp.Text`,
     `TotalTokens` = their sum, `Estimated = true`.
   - Assign the result to the `obs.Metrics.Tokens` of the metrics struct that
     17-01 placed on `Diagnostics`.
3. Add a small helper in `rag/ask.go` (or reuse one) to flatten a
   `generate.Request` to a single string for estimation ‚Äî system prompt plus
   each message `Content`, newline-joined.
4. Tests in `rag` (extend `rag/instrument_test.go` or a new
   `rag/tokens_test.go`):
   - A scripted model returning a `Response` with `Usage` set ‚Üí
     `answer.Diagnostics.Metrics.Tokens` equals the reported usage and
     `Estimated == false`.
   - A scripted model returning `Usage{}` (the default) ‚Üí `Tokens` is a
     positive estimate with `Estimated == true` and
     `TotalTokens == PromptTokens + CompletionTokens`.
5. Confirm the existing scripted model(s) used across the suite still
   compile and pass with the new `Response` field (they leave `Usage` zero).
6. Run every command in `<verify>`.

Out of scope:
- A real provider adapter that reports usage ‚Äî adapters live in
  `llm-agent-providers`; this slice only opens the seam.
- Embedding-token accounting ‚Äî the import flow's cost is its `Calls.Embed`
  count (17-01); per-embedding token estimation is not in scope.
- `otelrag` emission of the token metric ‚Äî that is 17-03.
</tasks>
