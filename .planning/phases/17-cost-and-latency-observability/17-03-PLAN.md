<objective>
`otelrag` emits spans only — it has no metrics. This slice delivers
RAG-OBS-02: the `otelrag.Wrapper` emits RED metrics (request counter, error
counter, duration histogram) and cost metrics (token counter) for every
`Import`, `Retrieve`, and `Ask`, derived from the `obs.Metrics` that
Phases 17-01/02 added to the RAG SDK.
</objective>

<context>
- `otelrag/otelrag.go` — `Config{TracerProvider}`, `Wrapper{inner, tracer}`,
  `Wrap(sys, ...Config) *Wrapper`, and `Import`/`Retrieve`/`Ask` methods,
  each opening a span. `Config.tracerProvider()` falls back to a noop tracer.
- `otelmetrics/otelmetrics.go` is the repo's metrics template: instruments
  built from `mp.Meter(name)` with a `noop.NewMeterProvider()` fallback;
  `Int64Counter` / `Int64Histogram` / `Float64Histogram`.
- The RAG SDK now exposes (via Phases 17-01/02, reached through `go.work` —
  see "Cross-repo verification" below):
  - `rag.Answer.Diagnostics.Metrics obs.Metrics` —
    `{TotalDuration, Stages []StageTiming, Calls, Tokens}`.
  - `ingest.ImportResult.Metrics obs.Metrics`.
  - `retrieve.Trace.Metrics` (carried via the `OnRetrieve` observer).
- `obs.TokenUsage` = `{PromptTokens, CompletionTokens, TotalTokens int;
  Estimated bool}`.
- `Wrap` returns `*Wrapper` (no error). Keep that signature — build
  instruments with a noop fallback so creation errors degrade silently.
- See `17-RESEARCH.md` Decision 5.
</context>

<tasks>
1. Extend `otelrag/otelrag.go` `Config` with `MeterProvider
   apimetric.MeterProvider` and a `meterProvider()` accessor falling back to
   `metric/noop.NewMeterProvider()` — mirroring `tracerProvider()`.
2. Create `otelrag/metrics.go`:
   - RAG-local metric name constants:
     `MetricRequests = "rag.requests"`, `MetricErrors = "rag.errors"`,
     `MetricDuration = "rag.operation.duration"`,
     `MetricTokens = "rag.tokens"`.
   - Metric attribute keys: `AttrOperation = "rag.operation"`,
     `AttrStage = "rag.stage"`, `AttrTokenKind = "rag.token.kind"`,
     `AttrError = "rag.error"`.
   - An `instruments` struct holding `requests apimetric.Int64Counter`,
     `errors apimetric.Int64Counter`, `duration apimetric.Float64Histogram`,
     `tokens apimetric.Int64Counter`.
   - `newInstruments(meter) instruments` — builds each; on any build error
     substitute the corresponding `noop` instrument so the result is always
     usable and `Wrap` need not return an error.
   - `recordOp(ctx, op string, elapsed time.Duration, err error)` — increment
     `requests` (attr `rag.operation`); on error increment `errors` and
     record `duration` with `rag.error=true`; on success record `duration`
     for the op; emit each `obs.Metrics.Stages` entry to `duration` tagged
     `rag.stage`.
   - `recordTokens(ctx, op string, t obs.TokenUsage)` — add `PromptTokens`
     and `CompletionTokens` to `tokens`, tagged `rag.operation` +
     `rag.token.kind` (`prompt`/`completion`).
3. Add an `instruments` field to `Wrapper`; build it in `Wrap` from
   `cfg.meterProvider().Meter(instrumentationName)`.
4. In `Wrapper.Import`/`Retrieve`/`Ask` (`otelrag/otelrag.go`):
   - Record `start := time.Now()` at entry; compute `elapsed` once the inner
     call returns.
   - Call `recordOp` with the operation name and the error (nil on success).
     Pass the inner `obs.Metrics` for per-stage emission: `Ask` uses
     `ans.Diagnostics.Metrics`; `Import` uses `res.Metrics`; `Retrieve` has
     no metrics on its return — emit the op-level RED only (documented).
   - On a successful `Ask`, call `recordTokens` with
     `ans.Diagnostics.Metrics.Tokens`. On a successful `Import`, the cost
     signal is `res.Metrics.Calls.Embed` — emit it to `tokens` tagged
     `rag.token.kind=embed_calls`, or skip if cleaner; keep `Ask` token
     emission the primary cost path.
   - Span behaviour is unchanged — metrics are added alongside, not instead.
5. Tests in `otelrag/otelrag_test.go`:
   - Build a `MeterProvider` from `sdkmetric.NewManualReader()` (pattern from
     `otelmetrics_test.go`), `Wrap` a `rag.System` (scripted model + hash
     embedder) with it, run `Ask`/`Import`, collect, and assert the
     instruments `rag.requests`, `rag.operation.duration`, and `rag.tokens`
     are present with an `rag.operation` attribute.
   - An error path (e.g. `Ask` with a model that errors, or empty query)
     emits `rag.errors`.
   - `Wrap` with no `MeterProvider` (noop) — `Ask` still succeeds and emits
     no panic (noop instruments).
6. Cross-repo verification — run via a temporary `go.work` (see `<verify>`):
   `otelrag` now references RAG-SDK fields that exist only in the untacked
   `llm-agent-rag` working tree. Do **not** add a committed `replace` (hard
   rule 3). Do **not** modify `otelrag/go.mod` or `go.sum`.
7. Run every command in `<verify>`.

Out of scope:
- Re-tagging `llm-agent-rag` and bumping `otelrag/go.mod`'s `require` — that
  is a milestone-close step (see "Carry-forward").
- Cost-in-currency conversion — `rag.tokens` is a token count; price
  mapping is a consumer concern.
- Metrics on the `otelrag.Observer()` event path — the `Wrapper` is the
  metrics path; the observer stays span-events only.
</tasks>
