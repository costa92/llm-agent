<objective>
16-01 added the `Judge` seam — the generation-quality scorer. This slice
assembles the full RAG Triad: a `TriadEvaluator` that runs a dataset through
the complete `Ask` pipeline, scores both retrieval (precision/recall/MRR/
grounding) and generation (groundedness/answer-relevance), produces one
combined `TriadResult` with a JSONL report and a summary, and adds a CI gate.
</objective>

<context>
- 16-01 added `eval.Judge`, `eval.JudgeRequest`, `eval.Judgement`.
- `eval.Evaluator` scores retrieval via a `Retriever`. Its metric helpers —
  `countMatches`, `firstGoldRank`, `anyOverlap` — are unexported in
  `eval/eval.go` and reusable from a sibling file in the same package.
- `eval.Metrics` (PrecisionAtK/RecallAtK/MRR/GroundingAtK/Examples/TopK) and
  `eval.Dataset`/`eval.Example` already exist and are JSON-tagged.
- `(*rag.System).Ask(ctx, question, rag.AskOptions) (rag.Answer, error)` runs
  retrieve + generate in one call. `rag.Answer` has `Text` and
  `Hits []store.Hit` (each hit's `Chunk` has `ID`, `DocID`, `Content`).
- `rag.AskOptions` embeds `Search rag.SearchOptions` (TopK, Namespace, ...).
- The existing CI gate `eval_test.go::TestSeedDatasetMeetsBaselineMetrics`
  builds a `rag.System` with `fakeModel` and a 4-example seed dataset.
- See `16-RESEARCH.md` Decision 3. No new dependency.
</context>

<tasks>
1. Create `eval/triad.go`:
   ```go
   // Asker runs the full retrieve+generate pipeline. *rag.System satisfies it.
   type Asker interface {
       Ask(ctx context.Context, question string, opts rag.AskOptions) (rag.Answer, error)
   }

   type GenerationMetrics struct {
       MeanGroundedness    float64 `json:"mean_groundedness"`
       MeanAnswerRelevance float64 `json:"mean_answer_relevance"`
       Examples            int     `json:"examples"`
   }

   type TriadExampleResult struct {
       Example      Example   `json:"example"`
       Answer       string    `json:"answer"`
       RetrievedIDs []string  `json:"retrieved_ids"`
       Judgement    Judgement `json:"judgement"`
   }

   type TriadResult struct {
       Dataset    Dataset              `json:"dataset"`
       Retrieval  Metrics              `json:"retrieval"`
       Generation GenerationMetrics    `json:"generation"`
       PerExample []TriadExampleResult `json:"per_example"`
   }

   type TriadEvaluator struct {
       Asker   Asker
       Judge   Judge
       Options rag.AskOptions
   }
   ```
2. Implement `TriadEvaluator.Run(ctx, dataset Dataset) (TriadResult, error)`:
   - error if `Asker == nil` or `Judge == nil`; error if `dataset.TopK <= 0`.
   - per example: copy `Options`, set `Options.Search.TopK = dataset.TopK`,
     overlay `Options.Search.Namespace` with `ex.Namespace` when non-empty.
   - call `Asker.Ask`; on error wrap and return.
   - derive `retrievedIDs` (chunk IDs) and `retrievedDocs` (doc IDs) from
     `answer.Hits`; compute the same four retrieval metrics as `Evaluator`
     using the shared helpers (`countMatches`, `firstGoldRank`, `anyOverlap`).
   - build the judge context from `answer.Hits` chunk `Content`; call
     `Judge.Judge` with `{Query, Answer: answer.Text, Context}`.
   - accumulate groundedness / answer-relevance sums.
   - assemble `TriadResult`: `Retrieval` (mean retrieval metrics, same
     formulas as `Evaluator.Run`), `Generation` (means over examples),
     `PerExample`.
3. Add report output to `eval/triad.go`:
   - `WriteJSONL(w io.Writer, r TriadResult) error` — writes one JSON line
     per `TriadExampleResult`, then a final summary line
     `{"summary":{...retrieval+generation...}}`.
   - `(TriadResult).Summary() string` — a compact human-readable scoreboard
     (dataset name, example count, the four retrieval metrics, the two
     generation metrics).
4. Create the CI gate `eval/triad_test.go`:
   - a deterministic stub `Judge` — e.g. groundedness = fraction of context
     passages sharing a word with the answer, answer-relevance = 1 if the
     answer shares a word with the query else 0. Deterministic, no model.
   - build a `rag.System` with `fakeModel` + the seed corpus (reuse the
     shape from `eval_test.go`), run `TriadEvaluator` on the seed dataset.
   - assert: retrieval metrics still meet the existing thresholds
     (`minPrecision/minRecall/minMRR/minGrounding`); `Generation.Examples`
     equals the dataset size; both generation means are within `[0,1]`.
   - assert `WriteJSONL` produces one line per example plus a summary line,
     and `Summary()` is non-empty.
5. Run every command in `<verify>`.

Out of scope:
- Replacing or modifying the retrieval-only `Evaluator` / its CI gate — both
  stay; `TriadEvaluator` is additive.
- Live LLM-as-judge in CI (no model available) — the gate uses the stub
  judge; `LLMJudge` quality is exercised by 16-01's unit tests.
- A CLI or service entry point for running evals (deferred milestone work).
</tasks>
