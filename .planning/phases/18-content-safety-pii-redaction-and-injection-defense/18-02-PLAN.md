<objective>
Retrieved chunk text is concatenated into the model prompt with no
inspection — a poisoned chunk is live prompt input. This slice adds
RAG-SEC-02: a `guard` injection scanner and a fail-safe sanitize step wired
into `Ask` so suspicious retrieved content is neutralized or dropped
**before** prompt assembly.
</objective>

<context>
- `guard` package exists from 18-01 (`guard/redact.go`).
- `rag/ask.go` `Ask` — after retrieve → rerank → pack, `packedHits` flow
  into `tpl.Render(ctx, prompt.RenderContext{Hits: packedHits})`. The
  sanitize step must run on `packedHits` before `Render`.
- `packedIDs`/`droppedIDs` and the `Trace`/`Diagnostics` assembly already
  exist in `Ask`; 17-01 added `obs.Metrics` plumbing there.
- `store.Hit` has `Chunk.ID` and `Chunk.Content`.
- `rag.Diagnostics` is the home for structured findings (it already carries
  `RerankScores`).
- `guard` stays string-oriented — it must not import `store`.
- See `18-RESEARCH.md` Decision 3.
</context>

<tasks>
1. Create `guard/inject.go` (package `guard`):
   ```go
   type InjectionVerdict struct { Suspicious bool; Patterns []string }
   type InjectionScanner interface { Scan(text string) InjectionVerdict }

   type InjectionPattern struct { Name string; Pattern *regexp.Regexp }
   type PatternScanner    struct { Patterns []InjectionPattern }

   type SanitizeMode int
   const ( Neutralize SanitizeMode = iota; Drop )
   ```
   - `PatternScanner.Scan(text)` returns `Suspicious = true` with the names
     of every matched pattern. The zero `PatternScanner` flags nothing.
   - `NewPatternScanner() PatternScanner` — built-in case-insensitive
     patterns for well-known injection phrasings: instruction-override
     ("ignore (all )?previous instructions", "disregard the above"),
     role-override ("you are now", "new instructions:"), and system-prompt
     exfiltration ("reveal/print your (system) prompt"). Compile with
     `(?i)`.
   - `Neutralize(text string) string` — wrap `text` in explicit markers
     declaring it untrusted data that must be treated strictly as data and
     never as instructions, so a neutralized chunk cannot act as a live
     instruction.
2. Add to `rag.Options` (`rag/options.go`): `InjectionScanner
   guard.InjectionScanner` and `SanitizeMode guard.SanitizeMode`.
3. Add `injectionScanner guard.InjectionScanner` and `sanitizeMode
   guard.SanitizeMode` fields to `System` (`rag/system.go`); set them from
   `opts` in `New` (nil scanner = off; `SanitizeMode` zero value =
   `guard.Neutralize`).
4. Add to `rag` an `InjectionFinding` struct:
   ```go
   type InjectionFinding struct {
       ChunkID  string
       Patterns []string
       Action   string // "neutralized" or "dropped"
   }
   ```
   and an `InjectionFindings []InjectionFinding` field on `Diagnostics`.
5. In `rag/ask.go` `Ask`, after packing and **before** `tpl.Render`: if
   `s.injectionScanner != nil`, scan each `packedHit`'s `Chunk.Content`.
   For a suspicious hit:
   - `guard.Drop` → exclude the hit from the slice passed to `Render`
     (and from `packedIDs`/citations), record an `InjectionFinding` with
     `Action: "dropped"`.
   - `guard.Neutralize` → replace that hit's `Chunk.Content` with
     `guard.Neutralize(content)` in the slice passed to `Render`, record an
     `InjectionFinding` with `Action: "neutralized"`.
   Non-suspicious hits pass through unchanged. Assign the findings to
   `Diagnostics.InjectionFindings`.
   - Keep the change additive: when `s.injectionScanner == nil`, `Ask`
     behaves exactly as before.
6. Tests `guard/inject_test.go`:
   - `NewPatternScanner().Scan` on text containing a known injection phrase
     → `Suspicious` true with the matched pattern name(s).
   - clean text → `Suspicious` false, `Patterns` empty.
   - `Neutralize(text)` → output contains the original text and the
     untrusted-data markers; output differs from input.
7. Tests in `rag` (e.g. `rag/inject_test.go`, package `rag`):
   - Import a clean chunk and a chunk whose content contains an injection
     phrase. `Ask` with `Options{InjectionScanner: guard.NewPatternScanner()}`
     and default mode → the poisoned chunk's content in the rendered prompt
     (`answer.Prompt`) is neutralized; `Diagnostics.InjectionFindings` has
     one entry with `Action: "neutralized"`.
   - Same with `SanitizeMode: guard.Drop` → the poisoned chunk is absent
     from `answer.Prompt` and `answer.Citations`; the finding's `Action` is
     `"dropped"`.
   - `Ask` with no scanner → poisoned content reaches the prompt unchanged
     (opt-in proof) and `InjectionFindings` is empty.
8. Run every command in `<verify>`.

Out of scope:
- PII redaction — that is 18-01.
- Scanning the user's question itself — the requirement scopes this to
  retrieved chunks before prompt assembly.
- A model-based injection classifier — `PatternScanner` is rule-based.
</tasks>
