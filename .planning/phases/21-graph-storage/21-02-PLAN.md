<tasks>
1. Extend `postgres.Store.Migrate` with two idempotent `CREATE TABLE IF NOT
   EXISTS` statements (namespaced; provenance as a `text[]` column):
   - `entities` — id, namespace, name, type, description, source_chunk_ids
     `text[]`, metadata `jsonb`; primary key `(namespace, id)`.
   - `relations` — id, namespace, source, target, relation, description,
     source_chunk_ids `text[]`, weight; primary key `(namespace, id)`.
   - Indexes: an index on `(namespace, source)` and `(namespace, target)`
     for traversal; one on `(namespace, lower(name))` for `FindEntities`.
   Keep the existing migration's style (idempotent, `isSafeIdent` for any
   configurable identifier).
2. Create `postgres/graph.go` — `postgres.Store` methods implementing
   `store.GraphStore`:
   - `UpsertGraph` — `INSERT ... ON CONFLICT (namespace, id) DO UPDATE`
     union-merging description and `source_chunk_ids` (array union);
     entities and relations in one transaction.
   - `RemoveGraphBySource(chunkIDs)` — remove the given ids from every
     row's `source_chunk_ids` (`array_remove` / anti-join), then
     `DELETE` rows whose `source_chunk_ids` became empty (GC).
   - `Neighborhood(seedIDs, depth)` — a `WITH RECURSIVE` CTE walking
     `relations` from the seeds (both directions), bounded by an explicit
     depth predicate (`depth` clamped to `[0, 2]` — KG-7) **and** a row
     `LIMIT` (the explosion guard); return `graph.Subgraph` with the
     reached entities, their relations, and the `Depth` map.
   - `FindEntities(names)` — `SELECT ... WHERE namespace = $1 AND
     lower(name) = ANY($2)`.
   - Compile-time `var _ store.GraphStore = (*Store)(nil)`.
3. Add `TestPostgresGraphConformance` to
   `postgres/postgres_conformance_test.go` — env-gated, reusing
   `openTestPool`/`newTableStore`, running `storetest.RunGraphConformance`.
4. Run every command in `<verify>`.

Out of scope:
- `Import` wiring — 21-03.
- Graph retrieval — Phase 22.
- Community detection / global search — v0.8.
</tasks>
