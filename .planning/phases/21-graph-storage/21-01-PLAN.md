<objective>
Open Phase 21 with the graph-storage seam: a `store.GraphStore` optional
capability interface, the `graph.Subgraph` traversal result, a pure-stdlib
in-memory implementation, and a shared `RunGraphConformance` suite. This is
the first half of RAG-GRAPH-03.
</objective>

<context>
- `store.LexicalSearcher` (`store/store.go`) is the optional-capability
  precedent — an interface a `Store` *may* implement, type-asserted by
  consumers with graceful fallback. `GraphStore` copies it.
- `graph` package (Phase 20): `Entity`, `Relation`, `Graph`,
  `Canonicalize`. `graph` imports only stdlib + `generate` — so `store`
  may import `graph` with no cycle.
- `store.NewInMemoryStore` is the default in-memory `store.Store`; the
  in-memory `GraphStore` is added alongside it (same package `store`).
- `store/storetest` — shared conformance: a `Factory func(t) store.Store`,
  `RunConformance` / `RunLexicalConformance` run `t.Run` subtests.
- See `21-RESEARCH.md` Decisions 1, 2, 3, 5.
</context>

<tasks>
1. Add `Subgraph` to `graph/graph.go`:
   ```go
   // Subgraph is the result of a neighborhood traversal: the reached
   // entities, the relations among them, and each entity's hop distance
   // from the nearest seed (seeds = 0).
   type Subgraph struct {
       Entities  []Entity
       Relations []Relation
       Depth     map[string]int
   }
   ```
2. Add the `GraphStore` interface to `store/store.go` (next to
   `LexicalSearcher`; add the `graph` import):
   ```go
   // GraphStore is an optional capability a Store may implement to persist
   // and traverse an entity/relation graph. Consumers type-assert for it
   // and degrade gracefully when a store does not implement it.
   type GraphStore interface {
       UpsertGraph(ctx context.Context, namespace string, g graph.Graph) error
       RemoveGraphBySource(ctx context.Context, namespace string, chunkIDs []string) error
       Neighborhood(ctx context.Context, namespace string, seedIDs []string, depth int) (graph.Subgraph, error)
       FindEntities(ctx context.Context, namespace string, names []string) ([]graph.Entity, error)
   }
   ```
3. Create `store/graph.go` — the in-memory `GraphStore` implementation on
   the in-memory store type:
   - per-namespace adjacency state: `map[ns]` → entities keyed by ID, and
     a relation adjacency index keyed by entity ID. Guard with the store's
     existing mutex (or a dedicated one) — keep it concurrency-safe like
     the rest of `InMemoryStore`.
   - `UpsertGraph` — **union-merge**: merge incoming entities by `ID`
     (descriptions concatenated-deduped, `SourceChunkIDs` unioned),
     relations by `ID`; new IDs are added.
   - `RemoveGraphBySource(chunkIDs)` — remove those chunk IDs from every
     entity/relation `SourceChunkIDs`; an entity or relation left with
     **empty** provenance is deleted (GC). Deterministic.
   - `Neighborhood(seedIDs, depth)` — breadth-first from the seeds over
     relations (treat edges as undirected for reachability); `depth` is
     clamped to `[0, 2]` (KG-7 hard cap); each hop expands at most
     `maxFanout` neighbors per entity (a documented package constant).
     Return a `graph.Subgraph` with the reached entities, the relations
     among reached entities, and the `Depth` map (seeds 0).
   - `FindEntities(names)` — return entities whose name matches any of
     `names` (normalized exact-match, same normalization as
     `graph.Canonicalize`).
   - Compile-time `var _ store.GraphStore = (*InMemoryStore)(nil)` (use the
     real in-memory store type name).
4. Add `RunGraphConformance(t, factory Factory)` to
   `store/storetest/storetest.go` — `t.Run` subtests, type-asserting the
   factory's `store.Store` to `store.GraphStore` (skip with a clear message
   if unimplemented, mirroring `RunLexicalConformance`):
   - `UpsertGraph` then `Neighborhood` from a seed → reaches its neighbors;
   - `Neighborhood` depth is bounded — depth 5 requested never returns
     entities more than 2 hops out;
   - `UpsertGraph` twice union-merges (provenance accumulates, no dup);
   - `RemoveGraphBySource` drops a chunk's contributions and GCs an entity
     left with no provenance, keeps one still referenced;
   - `FindEntities` resolves by name.
5. Wire `RunGraphConformance` for the in-memory store in
   `store/graph_test.go` (or the existing in-memory conformance test file).
6. Run every command in `<verify>`.

Out of scope:
- The `postgres` `GraphStore` — 21-02.
- Wiring `Import` to persist the graph — 21-03.
- Graph retrieval — Phase 22.
</tasks>
