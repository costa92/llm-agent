<objective>
RAG-ECO-02 says "both repos ship tests and CI gates that prevent
contract drift between standalone RAG and the core compatibility
layer." Today the contract is implicit: `go mod tidy` or
`go test ./rag/...` in the core happens to surface drift, but only
when someone runs it. A rename in standalone can land and stay
silent for weeks.

This slice ships two contract tests — one in each repo — that
reference the cross-repo surface by name. Any rename or removal
of a contract symbol fails compilation, which means `go test`
itself becomes the gate. No new workflow files are needed; the
existing test runs do the work.

GitHub Actions / cross-repo workflow plumbing is intentionally
out of scope. CLAUDE.md still says to leave `.github/workflows/`
alone until refactor day, and the contract tests are themselves
sufficient to satisfy RAG-ECO-02 — they fire in every PR's test
run.
</objective>

<context>
- The core's `rag/` facade adapts standalone types via four adapter
  structs (`splitterAdapter`, `embedderAdapter`, `storeAdapter`,
  `modelAdapter`). Each adapter implements a standalone interface
  by name. If standalone renames or changes those interfaces, the
  adapters fail to compile.
- The standalone packages consumed by the core today are:
  `ingest` (Document, Chunk, Splitter), `embed` (Vector, Embedder),
  `store` (Store, StoredChunk, Query, Hit, Filter, Stats,
  ErrNotFound, ErrDimensionMismatch), `generate` (Model, Request,
  Response).
- The core's facade exports:
  - types: `RAGSystem`, `Options`, `SearchOptions`, `Document`,
    `SearchHit`, `VectorStore`, `StoreStats`, `InMemoryStore`
  - constructors: `New`, `NewInMemoryStore`, `AsTool`
  - methods: `(*RAGSystem).AddText/.Search/.Ask/.Remove/.Stats`
  - errors: `ErrEmptyQuery`, `ErrLLMRequired`, `ErrStoreNotFound`,
    `ErrDimMismatch`
- Tests that merely *call* these symbols already exist on both
  sides. What's missing is **explicit named references** that
  cause a compile error on rename/removal — adapters reference
  some but not all of the contract surface (e.g. eval / feedback
  / observer types aren't part of the core facade's contract but
  the core consumes them transitively via `rag.Trace`).
- The new contract files declare unused-but-referenced variables
  using the `_ = symbol` pattern to keep the surface explicit
  without producing dead code warnings.
</context>
