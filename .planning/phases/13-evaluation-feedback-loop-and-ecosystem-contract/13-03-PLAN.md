<objective>
RAG-OPS-03 says "production failures can be captured as regression
cases and fed back into retrieval evaluation." 13-01 shipped the
eval framework and the JSONL format. 12-03 shipped the observer
hook. This slice closes the loop: a tiny `feedback` package that
makes it one method call to capture a flagged Ask as a JSONL
example, with the same shape `eval.LoadJSONL` consumes.

Design is deliberately minimal — no automatic "capture every ask"
helper, no separate miss queue, no scoring pipeline. The user
decides which Asks are misses (in their own OnAsk callback) and
calls `Capture` on the ones that matter. Everything else they need
already exists in `rag.Trace` and `eval.Example`.
</objective>

<context>
- `eval.Example` from 13-01 has JSON tags: `query`, `namespace`,
  `gold_doc_ids`, `gold_chunk_ids`, `notes`. The JSONL format the
  loader reads is one Example per line.
- `rag.Trace.Question` and `rag.Trace.Namespace` map directly onto
  `Example.Query` and `Example.Namespace`. The gold information
  (doc IDs, chunk IDs) is what the user supplies after a human (or
  automated) flag.
- `rag.Observer.OnAsk` fires from inside `*System.Ask`. That call
  site is synchronous, but Ask itself may be invoked from many
  goroutines. The Recorder must serialize file writes.
- No new dependencies. `encoding/json`, `io`, `os`, `sync`, and
  `bufio` are sufficient.
</context>

<tasks>
1. New package `feedback/`:
   - `feedback/feedback.go`:
     ```go
     // Recorder appends eval.Example values to an underlying io.Writer
     // in JSONL form. Safe for concurrent use.
     type Recorder struct {
         mu  sync.Mutex
         out io.Writer
     }

     func NewRecorder(out io.Writer) *Recorder

     // OpenFile opens (or creates) path for append-write and returns a
     // Recorder plus a Close func. The caller is responsible for
     // calling Close when done.
     func OpenFile(path string) (*Recorder, func() error, error)

     // Capture writes ex as a single JSON line, terminated by '\n'.
     func (r *Recorder) Capture(ex eval.Example) error

     // BuildExample maps a rag.Trace + caller-supplied gold info into an
     // eval.Example. Query and Namespace come from the trace.
     func BuildExample(trace rag.Trace, goldDocIDs, goldChunkIDs []string, notes string) eval.Example
     ```
   - Capture writes are line-buffered: marshal Example, append '\n',
     write under the mutex, flush. Multiple goroutines calling
     Capture concurrently must produce a well-formed JSONL file.
   - OpenFile uses `os.O_APPEND|os.O_CREATE|os.O_WRONLY`, mode
     `0o600`. Returns a closer that closes the file (and only the
     file — the Recorder itself has no resources).
2. `feedback/feedback_test.go`:
   - `TestBuildExampleMapsTrace` — Trace fields land in the right
     Example slots; gold inputs round-trip
   - `TestRecorderCaptureWritesJSONL` — capture two examples,
     assert two lines, each valid JSON, fields preserved
   - `TestRecorderCaptureConcurrent` — N goroutines each Capture
     once; assert N lines, each parseable, no truncation
   - `TestRoundTripObserverToLoadJSONL` — end-to-end:
     - construct `*rag.System` with an `OpenFile`-backed
       Recorder wired into `Observer.OnAsk` (filtered: only
       capture when the query contains a magic substring, to
       prove the opt-in shape)
     - run two Asks: one captured, one ignored
     - close the Recorder file
     - call `eval.LoadJSONL` on the file
     - assert the loaded Dataset contains exactly the captured
       example, with the expected Query / Namespace / gold IDs
3. Verify per `<verify>`.

Out of scope:
- A separate "miss queue" type. Users buffer in their callback if
  they want batching.
- Capturing every Ask automatically. The package surface
  deliberately makes the user write the filter.
- Promotion logic (which captured cases become CI-gated). That's a
  user policy choice, not a library concern.
- Auto-deduplication of identical captures. The JSONL format
  tolerates duplicates; consumers can dedupe at load time if they
  want.
</tasks>
