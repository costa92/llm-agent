<objective>
Add RAG-AGENT-02: a self-correcting retrieval loop. A new `agentic`
package's `CorrectiveAsker` runs the full retrieve→generate pipeline,
judges the answer's groundedness with the Phase 16 `eval.Judge` signal,
and — when grounding is low — reformulates the query and retries, up to a
bounded cap.
</objective>

<context>
- `eval.Asker` — `Ask(ctx, question string, opts rag.AskOptions)
  (rag.Answer, error)`; `*rag.System` satisfies it.
- `eval.Judge` — `Judge(ctx, JudgeRequest{Query, Answer string,
  Context []string}) (Judgement{Groundedness, AnswerRelevance float64,
  Rationale string}, error)`. `Groundedness` is the Phase 16 grounding
  signal, in `[0,1]`.
- `rag.Answer` carries `Text` and `Hits []store.Hit` (each `Hit.Chunk.Content`).
- `agentic` is a new package — it imports `eval`, `rag`, `generate`,
  `store`, `context`, `errors`, `strings`. No cycle (`eval` imports `rag`;
  `agentic` sits above both).
- The project's mock discipline: LLM-backed types are tested with a
  scripted model; orchestrators with deterministic stubs.
- See `19-RESEARCH.md` Decision 2.
</context>

<tasks>
1. Create `agentic/correct.go` (package `agentic`):
   - `QueryReformulator` interface:
     ```go
     type QueryReformulator interface {
         Reformulate(ctx context.Context, question string, prev rag.Answer) (string, error)
     }
     ```
   - `LLMReformulator struct { Model generate.Model }` — `Reformulate`
     prompts the model with the original question and the previous
     (poorly-grounded) answer, asking for a rewritten retrieval query;
     trims the response; an empty response → return `question` unchanged;
     `Model == nil` → return `question` unchanged.
   - `Attempt struct { Query string; Groundedness, AnswerRelevance float64 }`.
   - `Result struct { Answer rag.Answer; Attempts []Attempt; Corrected bool }`
     (`Corrected` = more than one attempt ran).
   - `CorrectiveAsker struct { Asker eval.Asker; Judge eval.Judge;
     Reformulator QueryReformulator; MinGrounding float64; MaxRetries int }`.
   - Export `ErrAskerRequired`, `ErrJudgeRequired`, `ErrReformulatorRequired`.
   - `AskWithCorrection(ctx, question string, opts rag.AskOptions)
     (Result, error)`:
     - nil `Asker`/`Judge`/`Reformulator` → the matching error.
     - resolve defaults: `MinGrounding <= 0` → `0.5`; `MaxRetries <= 0` → `2`.
     - loop `i` from `0` to `maxRetries` inclusive (at most
       `maxRetries + 1` attempts):
       - `answer, err := Asker.Ask(ctx, query, opts)` — `query` starts as
         `question`.
       - `j, err := Judge.Judge(ctx, eval.JudgeRequest{Query: question,
         Answer: answer.Text, Context: <answer.Hits contents>})` — judge
         always with the **original** `question`.
       - append an `Attempt{query, j.Groundedness, j.AnswerRelevance}`;
         track the best answer by `Groundedness`.
       - if `j.Groundedness >= minGrounding` → break (good enough).
       - if `i == maxRetries` → break (cap reached).
       - else `query, err = Reformulator.Reformulate(ctx, question, answer)`.
     - return `Result{Answer: <best by groundedness>, Attempts,
       Corrected: len(Attempts) > 1}`.
   - `Ask(ctx, question string, opts rag.AskOptions) (rag.Answer, error)` —
     calls `AskWithCorrection` and returns `Result.Answer`; this satisfies
     `eval.Asker`, so a `CorrectiveAsker` composes inside a
     `TriadEvaluator`. Add a compile-time `var _ eval.Asker =
     CorrectiveAsker{}`.
2. Tests `agentic/correct_test.go`:
   - deterministic stubs: a `stubAsker` returning answers keyed by query;
     a `stubJudge` returning a groundedness keyed by answer text; a
     `stubReformulator` returning a fixed next query.
   - correction succeeds: the original query yields a low-grounding answer,
     the reformulated query a high-grounding one → `Result.Answer` is the
     high-grounding answer, `len(Attempts) == 2`, `Corrected == true`.
   - cap respected: the asker always yields low grounding, `MaxRetries: 2`
     → `len(Attempts) == 3`, the returned answer is the best attempt,
     `Corrected == true`.
   - no correction needed: a high-grounding first answer →
     `len(Attempts) == 1`, `Corrected == false`.
   - nil `Asker` / nil `Judge` / nil `Reformulator` → the matching error.
   - `LLMReformulator.Reformulate` with a scripted `generate.Model` →
     returns the model's reformulation; with a `nil` model → returns the
     original question.
3. Run every command in `<verify>`.

Out of scope:

- Surfacing the correction trace on `rag.Diagnostics` — `rag` cannot import

  `AskWithCorrection`.
- A non-LLM heuristic reformulator — `LLMReformulator` is the deliverable;
  the `QueryReformulator` seam allows others.
</tasks>
