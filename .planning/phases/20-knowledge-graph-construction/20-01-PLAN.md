<objective>
Open the v0.7 GraphRAG milestone with the `graph` package: the `Entity` /
`Relation` / `Graph` types, an `EntityExtractor` seam, and an
`LLMEntityExtractor` that extracts entities and typed relations from a
chunk's text by prompting a `generate.Model`. This is RAG-GRAPH-01 — the
construction primitives, standalone and fully unit-tested.
</objective>

<context>
- New `graph` package — peer of `retrieve`/`rerank`/`eval`. Imports only
  the stdlib plus `generate` (for `LLMEntityExtractor`).
- `advanced/llm.go` is the prompt+parse template: `ExpandQuery` builds a
  `generate.Request`, calls `model.Generate`, and parses `resp.Text`
  line-by-line (`strings.Split` on `\n`, `TrimSpace`, `TrimLeft` markers,
  drop empties). Copy this shape.
- `generate.Model`: `Generate(ctx, Request) (Response, error)`;
  `Request{SystemPrompt string, Messages []Message, Metadata}`;
  `Message{Role, Content}`; `Response{Text string, Usage}`.
- Determinism discipline: an LLM-backed type is unit-tested with a scripted
  `generate.Model` returning canned text — never a live call.
- See `20-RESEARCH.md` Decisions 1 and 2, and
  `.planning/research/v0.7-graphrag-SUMMARY.md` §6.
</context>

<tasks>
1. Create `graph/graph.go` (package `graph`):
   ```go
   // Entity is a node — a salient real-world thing named in the corpus.
   type Entity struct {
       ID             string         // canonical id; empty until Canonicalize
       Name           string
       Type           string         // free-form: "person", "org", ...
       Description    string
       SourceChunkIDs []string        // provenance
       Metadata       map[string]any
   }

   // Relation is a directed, typed edge.
   type Relation struct {
       ID             string
       Source, Target string         // entity names pre-Canonicalize, ids after
       Relation       string
       Description    string
       SourceChunkIDs []string
       Weight         float64
   }

   // Graph is a canonicalized entity/relation set.
   type Graph struct {
       Entities  []Entity
       Relations []Relation
   }

   // EntityExtractor turns one chunk's text into raw (pre-canonical)
   // graph primitives. Implementations may be LLM-backed or deterministic.
   type EntityExtractor interface {
       Extract(ctx context.Context, chunkID, text string) ([]Entity, []Relation, error)
   }
   ```
2. Create `graph/extract.go`:
   - `var ErrEntityExtractorModelRequired = errors.New("graph: entity extractor requires a generate.Model")`.
   - `LLMEntityExtractor struct { Model generate.Model }` implementing
     `EntityExtractor`.
   - A fixed system prompt instructing the model to extract **salient**
     entities and relations (not every noun) and emit one record per line,
     pipe-delimited:
     ```
     ENTITY | name | type | description
     RELATION | source-name | target-name | relation | description
     ```
     with no commentary. (Salience guidance pre-empts graph bloat — RESEARCH
     pitfalls.)
   - `Extract`: nil `Model` → `ErrEntityExtractorModelRequired`. Build a
     `generate.Request` (system prompt + the chunk text as a user message),
     call `Generate`, parse `resp.Text` leniently:
     - split on `\n`, trim each line;
     - classify by the first pipe-field (case-insensitive `ENTITY` /
       `RELATION`); lines that match neither are skipped;
     - an `ENTITY` line needs ≥ name; missing type/description default to
       empty. A `RELATION` line needs ≥ source, target, relation.
     - too-few-fields or junk lines are dropped, never fatal.
   - Each extracted `Entity` gets `SourceChunkIDs: []string{chunkID}`,
     empty `ID` (Canonicalize assigns it — slice 20-02). Each `Relation`
     gets `SourceChunkIDs: []string{chunkID}`, `Source`/`Target` as the
     raw names from the line, `Weight: 1`.
3. Tests `graph/extract_test.go`:
   - a scripted `generate.Model` returning a clean multi-line extraction
     (2 entities, 1 relation) → assert the parsed entities/relations match,
     `SourceChunkIDs` carries the chunk id.
   - a scripted model returning output with junk lines, a prose preamble,
     an `ENTITY` line missing its description, and a `RELATION` line with
     too few fields → assert the lenient parser keeps the valid records and
     drops the malformed ones without error.
   - `LLMEntityExtractor{}` (nil model) → `ErrEntityExtractorModelRequired`.
4. Tests `graph/graph_test.go`: a minimal compile/shape test of the types
   (construct an `Entity`, `Relation`, `Graph`) — keep light; `Canonicalize`
   is tested in 20-02.
5. Run every command in `<verify>`.

Out of scope:
- `Canonicalize` (cross-chunk merge), the deterministic
  `DictionaryEntityExtractor`, and the `Import` wiring — all 20-02.
- Any `store`/`retrieve` integration — Phases 21-22.
</tasks>
