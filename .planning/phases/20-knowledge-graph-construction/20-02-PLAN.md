<tasks>
1. `graph/dictionary.go` — `DictionaryEntityExtractor`:
   ```go
   // DictionaryEntityExtractor is a deterministic, zero-LLM EntityExtractor:
   // it matches a caller-supplied gazetteer (term -> type) against chunk
   // text. It is the no-LLM default and the basis for reproducible tests.
   type DictionaryEntityExtractor struct {
       Terms map[string]string // term -> entity type
   }
   ```
   - `Extract`: case-insensitively find each gazetteer term in `text`;
     each distinct match → an `Entity{Name: <canonical term>, Type,
     SourceChunkIDs: [chunkID]}`. For every unordered pair of entities
     co-occurring in the chunk, emit a `Relation{Source, Target,
     Relation: "co-occurs", SourceChunkIDs: [chunkID], Weight: 1}`.
   - Deterministic ordering (sort terms) so output is stable.
2. `graph/canonicalize.go` — `Canonicalize`:
   ```go
   // Canonicalize merges raw per-chunk extractions into one Graph: entities
   // merge by (normalized name, type); relation endpoints resolve from
   // names to canonical entity IDs.
   func Canonicalize(entities []Entity, relations []Relation) Graph
   ```
   - Normalize a name: `strings.ToLower(strings.TrimSpace(name))`.
   - Merge entities keyed by `(normalizedName, Type)`: the canonical
     `Entity` keeps the first-seen `Name`/`Type`, gets a stable `ID`
     (e.g. `Type + ":" + normalizedName`), a concatenated-deduped
     `Description`, and unioned (deduped) `SourceChunkIDs`.
   - Resolve each relation's `Source`/`Target` (names) to canonical entity
     `ID`s via the same normalization; a relation whose endpoint matches no
     entity is dropped. Merge relations by `(Source, Target, Relation)`:
     unioned `SourceChunkIDs`, summed `Weight`, stable `ID`.
   - Deterministic output ordering (sort entities by `ID`, relations by
     `ID`).
3. Add `Graph *graph.Graph` to `ingest.ImportResult` (`ingest/types.go` —
   add the `graph` import).
4. Add `EntityExtractor graph.EntityExtractor` to `rag.Options`
   (`rag/options.go`); add an `entityExtractor` field to `System`
   (`rag/system.go`), set from `opts.EntityExtractor` in `New` (nil = off).
5. Wire extraction into `rag/import.go` `Import`:
   - When `s.entityExtractor != nil`: for each chunk produced by the
     splitter, call `s.entityExtractor.Extract(ctx, chunk.ID, chunk.Content)`
     and accumulate the raw entities/relations across the whole import.
   - After the document loop, `graph.Canonicalize(allEntities,
     allRelations)` and set `res.Graph = &g`.
   - When `s.entityExtractor == nil`, `res.Graph` stays nil and `Import`
     behaves exactly as before (verify this).
   - An `Extract` error aborts the import with a wrapped error (consistent
     with how `embed` errors abort).
6. Tests:
   - `graph/dictionary_test.go` — `DictionaryEntityExtractor.Extract` on
     text containing two gazetteer terms → two entities + one `co-occurs`
     relation; deterministic across runs.
   - `graph/canonicalize_test.go` — raw extractions of the same entity name
     from two different chunks → one canonical entity with both chunk ids
     in `SourceChunkIDs`; a relation by name → endpoints resolved to ids;
     a relation with an unknown endpoint → dropped.
   - `rag/graph_test.go` — `Import` with `Options{EntityExtractor:
     graph.DictionaryEntityExtractor{Terms: ...}}` of documents whose text
     contains gazetteer terms → `ImportResult.Graph` is non-nil with the
     expected canonical entities and provenance spanning chunks; `Import`
     with no extractor → `ImportResult.Graph` is nil and content is
     unaffected.
7. Run every command in `<verify>`.

</tasks>
